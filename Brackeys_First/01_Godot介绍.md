# 01 Godot 介绍

## 本节目标

- 完成 Godot 引擎的下载与安装
- 创建第一个 2D 项目
- 理解 Godot 的节点、场景与场景树核心概念
- 导入游戏资源并建立基础文件夹结构

## Godot 简介

- Godot 是一款开源、免费的游戏引擎，对初学者友好。
- 本系列使用 2D 模式进行开发，但所学概念可较容易地迁移到 3D。
- 脚本语言使用 GDScript，语法相对简单且与引擎结合紧密。

## 安装与项目创建

1. 访问 [Godot 官网](https://godotengine.org)，点击 **Download Latest**。
2. 下载 Godot Engine（轻量版，无需安装即可运行）。
3. 解压到一个容易记住的位置。
4. 双击运行 Godot，取消示例项目提示。
5. 点击 **新建项目**，命名项目（例如“第一个游戏”）。
6. 为项目创建独立文件夹，便于管理。
7. 点击 **创建并编辑**，进入 Godot 编辑器。
8. 项目默认打开 3D 视图，点击顶部 **2D** 按钮切换到 2D 模式。

## 资源准备与导入

- 游戏资源包括精灵（Sprites）、模型、纹理、音效、音乐等，统称为**资源（Assets）**。
- 使用他人资源时需注意许可协议：
  - **CC0**：完全免费，无需署名。
  - 其他许可需按要求署名或遵守限制。
- 建议初学者先用现成资源占位，熟悉后再尝试自制。

### 导入步骤

1. 在文件系统窗口中右键，创建以下文件夹：
   - `assets`：存放图片、音频、字体等资源。
   - `scripts`：存放脚本文件。
   - `scenes`：存放场景文件。
2. 将资源包中的子文件夹直接拖入 `assets` 目录。
3. Godot 会自动导入资源，导入后即可在项目中使用。

## 节点与场景

### 节点（Node）

- 节点是构成游戏的基本单元。
- 不同节点负责不同功能：显示图像、播放声音、处理物理碰撞等。
- 可以通过组合和扩展现有节点来构建复杂功能。

### 场景（Scene）

- 场景是将节点打包成的可复用模块。
- 一个场景可以是一个角色、一件武器、一个菜单，甚至整个关卡。
- 场景可以嵌套：在一个场景内部放入其他场景。
- 场景可复用：编辑一次，所有实例同步更新。

### 场景树（Scene Tree）

- 所有节点和场景在游戏中呈现为树状结构。
- 树的最顶层节点称为**根节点**。
- 通过场景树可以清晰管理游戏对象的层级关系。
