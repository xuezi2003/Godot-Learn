# 06 死亡机制 1.0 版

## 本节目标

- 限制摄像机，避免玩家掉落时镜头跟随
- 创建可复用的死亡区域场景
- 使用 Timer 节点实现延迟重启
- 学习节点路径与信号的使用

## 限制摄像机

1. 打开 `player` 场景，选中 `Camera2D` 节点。
2. 在 Inspector 中找到 **Limit** 选项。
3. 可以分别设置摄像机左侧、顶部、右侧和底部的位置限制。
4. 使用标尺工具（按 `R`）测量关卡底部位置。
5. 在 **Bottom** 中输入一个合适的值，例如 **120**。
6. 启用 **Position Smoothing**。
7. 运行游戏，玩家掉落时摄像机会保持在限制位置，不会跟随玩家坠出画面。

## 创建死亡区域场景

- 死亡区域不仅能用于跌落地图外，还能复用于尖刺、敌人等危险元素。
- 可复用的元素通常做成独立场景。

1. 新建场景，根节点选择 **Area2D**。
2. 将根节点重命名为 **Killzone**（死亡地带）。
3. 在 Collision 中将 **Collision Mask** 设置为第 2 层，只检测玩家。
4. **不添加碰撞形状**，以便在不同使用时自定义形状。
5. 保存到 `scenes` 文件夹，命名为 `killzone.tscn`。

## 在游戏场景中使用死亡区域

1. 在 `game` 场景中实例化 `killzone.tscn`。
2. 选中 `Killzone` 实例，添加子节点 **CollisionShape2D**。
3. Shape 选择 **WorldBoundaryShape2D**。
4. 使用移动工具（`W`）将死亡区域移动到关卡底部下方，远离正常游玩区域。

## 死亡区域脚本

1. 在 `killzone` 场景中选中 `Killzone` 节点。
2. 添加脚本，选择空模板，保存到 `scripts/killzone.gd`。
3. 连接 `body_entered` 信号，生成 `_on_body_entered(body)` 函数。
4. 在函数中添加打印：

```gdscript
func _on_body_entered(body):
    print("你死了")
```

5. 运行游戏并跳入死亡区域，输出窗口会显示“你死了”。

## 添加延迟重启

### 使用 Timer 节点

1. 在 `killzone` 场景中添加子节点 **Timer**。
2. 在 Inspector 中：
   - 将 **Wait Time** 设置为 **0.6** 秒。
   - 启用 **One Shot**，使其只触发一次，不循环。

### 获取计时器引用

1. 在脚本顶部，从场景树中拖动 `Timer` 节点到代码编辑器。
2. 释放时按住 `Ctrl` 键，会自动生成节点引用变量：

```gdscript
@onready var timer = $Timer
```

### 节点路径说明

- 路径指定了从一个节点到达另一个节点的方式。
- `$Timer` 表示从当前节点直接访问名为 `Timer` 的子节点。
- 这种引用方式简单且常用。

### 启动计时器

```gdscript
func _on_body_entered(body):
    print("你死了")
    timer.start()
```

### 计时器超时处理

1. 选中 `Timer` 节点，连接 `timeout` 信号。
2. 脚本中生成 `_on_timer_timeout()` 函数。
3. 在函数中重新加载当前场景：

```gdscript
func _on_timer_timeout():
    get_tree().reload_current_scene()
```

## 死亡流程

1. 玩家进入死亡区域 → 触发 `_on_body_entered`。
2. 打印死亡信息并启动计时器。
3. 计时器 0.6 秒后触发 `_on_timer_timeout`。
4. 重新加载当前场景，游戏重启。
