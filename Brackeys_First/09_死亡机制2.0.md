# 09 死亡机制 2.0 版

## 本节目标

- 添加死亡时的慢动作效果
- 在死亡时移除玩家碰撞体，使玩家掉出地图
- 理解 `Engine.time_scale` 的使用

## 死亡慢动作效果

1. 打开 `killzone` 场景中的脚本 `scripts/killzone.gd`。
2. 在 `_on_body_entered` 中打印死亡信息后，修改时间尺度：

```gdscript
func _on_body_entered(body):
    print("你死了")
    Engine.time_scale = 0.5
    timer.start()
```

- `Engine.time_scale` 控制游戏整体运行速度。
- 设置为 **0.5** 表示所有内容以半速运行。

### 恢复时间尺度

- 计时器超时重新加载场景时，需要将时间尺度恢复为默认值。
- 否则重启后游戏仍处于慢速状态。

```gdscript
func _on_timer_timeout():
    Engine.time_scale = 1.0
    get_tree().reload_current_scene()
```

## 移除玩家碰撞体

- 死亡时让玩家被“击飞”并掉出地图，视觉效果更酷。
- 可以通过移除玩家的碰撞体实现。

### 获取进入区域的物体

- `_on_body_entered(body)` 中的 `body` 参数就是进入死亡区域的物体。
- 由于死亡区域只检测第 2 层的玩家，`body` 即玩家节点。

### 删除碰撞形状

```gdscript
func _on_body_entered(body):
    print("你死了")
    Engine.time_scale = 0.5
    body.get_node("CollisionShape2D").queue_free()
    timer.start()
```

- `body.get_node("CollisionShape2D")` 获取玩家下的碰撞形状节点。
- `queue_free()` 将该节点从场景中移除。
- 玩家失去碰撞体后会继续下落，掉出地图。

## 完整死亡区域脚本

```gdscript
extends Area2D

@onready var timer = $Timer

func _on_body_entered(body):
    print("你死了")
    Engine.time_scale = 0.5
    body.get_node("CollisionShape2D").queue_free()
    timer.start()

func _on_timer_timeout():
    Engine.time_scale = 1.0
    get_tree().reload_current_scene()
```

## 注意事项

- `Engine.time_scale` 会影响游戏中的所有内容，包括计时器、动画和物理。
- 因此死亡延迟和重启时间也会变慢。
- 必须在场景重新加载前恢复 `time_scale`，否则下一局仍会处于慢动作。
