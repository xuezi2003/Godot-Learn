# 03 场景碰撞与子弹基础

## 为瓦片添加碰撞

1. 在 `game` 场景中选中 `GroundTileMapLayer`。
2. 在 Inspector 中点击 **Tile Set**。
3. 找到 **Physics Layers**，点击添加元素。
4. 在 TileSet 面板中选中静态瓦片图集，点击绘制按钮。
5. 展开属性编辑器，选择 **Physics Layer 0**。
6. 在编辑窗口中框选用作障碍物的瓦片，使其变蓝。
7. 动态瓦片同样操作，只需设置每一行第一列，后续动画帧会共用同一碰撞外形。

## 调整玩家碰撞尺寸

- 若角色无法进入一格宽的小道，说明碰撞外形过大。
- 在 `player` 场景中选中 `CollisionShape2D`，将 CircleShape2D 的 **Radius** 从 10 调整为 6。

## 地图边界空气墙

1. 在 `game` 场景中添加 **Node2D**，命名为 `WorldBoundaries`。
2. 在其下添加 **StaticBody2D**，命名为 `TopBoundary`。
3. 在边界节点下添加 **CollisionShape2D**。
4. Shape 选择 **SegmentShape2D**，拖动端点覆盖地图边缘。
5. 打开编辑器顶部的 **栅格吸附**，方便对齐。
6. 为地图的四条边分别创建边界节点。

> 不要直接复制已有的 StaticBody2D 节点，否则 CollisionShape2D 可能共享同一形状资源，调整时会互相干扰。

## 调试碰撞区域

- 打开 **Debug > Visible Collision Shapes**。
- 运行时可以看到蓝色的碰撞外形，便于调试和调整手感。

## 子弹需求分析

- 玩家按方向键向八个方向射击。
- 吃到强化道具后发射螺旋弹幕。
- 子弹命中敌人或墙壁后消失。

## Area2D 节点

| 节点 | 用途 |
|-----|------|
| Node2D | 只有位置、旋转、缩放，无物理和碰撞特化 |
| CharacterBody2D | 脚本控制移动，可与世界发生实体碰撞和滑动 |
| RigidBody2D | 完全由物理系统模拟，适合箱子、石头、被炸飞的杂物 |
| StaticBody2D | 不会移动的障碍物，如墙壁、柱子、地图边界 |
| Area2D | 范围检测，不处理物理碰撞，适合子弹命中、道具拾取、爆炸范围、受击检测 |

黄金规则：

- 只做范围检测，不需要物理模拟 → **Area2D**
- 自己控制移动，需要实体碰撞 → **CharacterBody2D**
- 完全交给物理系统 → **RigidBody2D**
- 不会动的障碍物 → **StaticBody2D**

## 创建 Bullet 场景

1. 在 `scenes` 目录下新建场景。
2. 根节点选择 **Area2D**。
3. 场景命名为 `bullet`。
4. 添加子节点：
   - **CollisionShape2D**：碰撞外形，Shape 选择 **CircleShape2D**，Radius 设为 3。
   - **Sprite2D**：渲染子弹纹理。
5. 将 Sprite2D 放到 CollisionShape2D 前面，方便观察碰撞外形。

### 使用 AtlasTexture 截取子弹纹理

1. 选中 Sprite2D，在 Inspector 中点击 **Texture > New AtlasTexture**。
2. 选择道具 UI 图片作为 Atlas。
3. 点击 **Edit Region**，选择自动裁剪模式。
4. 选中子弹对应的区域。

## 子弹脚本设计思路

- 将子弹的状态和生命周期封装在独立脚本中，不要交给玩家脚本管理。
- 子弹自己负责：飞行、碰撞检测、销毁。
- 对外只暴露简单的初始化接口。

### 关键代码结构

```gdscript
class_name Bullet
extends Area2D

const WORLD_COLLISION_MASK := 1

@export var speed: float = 400.0
@export var max_lifetime: float = 5.0

var direction: Vector2
var lifetime: float

func _ready() -> void:
    area_entered.connect(_on_area_entered)

func setup(fire_direction: Vector2) -> void:
    direction = fire_direction.normalized()

func _physics_process(delta: float) -> void:
    var start_position := global_position
    var end_position := start_position + direction * speed * delta

    if will_hit_world(start_position, end_position):
        queue_free()
        return

    global_position = end_position

    lifetime += delta
    if lifetime >= max_lifetime:
        queue_free()

func will_hit_world(from: Vector2, to: Vector2) -> bool:
    var query := PhysicsRayQueryParameters2D.create(from, to, WORLD_COLLISION_MASK)
    var result := get_world_2d().direct_space_state.intersect_ray(query)
    return not result.is_empty()

func _on_area_entered(other: Area2D) -> void:
    if other is Bullet:
        return
    queue_free()
```

### 要点说明

- **class_name**：给脚本定义全局可识别的类名，方便其他脚本直接引用 `Bullet` 类型。
- **WORLD_COLLISION_MASK**：物理层掩码，用于过滤子弹需要检测的碰撞层。值为 `1` 表示第 1 层，后续会深入讲解物理层。
- **_ready()**：节点实例化后自动调用，用于连接信号。
- **setup()**：由创建者（如玩家）调用，传入飞行方向。
- **area_entered 信号**：当 Area2D 与其他区域重叠时触发。
- **queue_free()**：标记对象在帧结束时删除，避免直接调用 `free()` 导致的时序和内存问题。
- **射线检测**：用于防止子弹速度过快时穿透薄障碍物或边界。
