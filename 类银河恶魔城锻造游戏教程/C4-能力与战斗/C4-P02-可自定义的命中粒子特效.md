# C4-P02：可自定义的命中粒子特效

> 本集在全局视觉特效工厂中加入”命中粒子”系统，通过自定义 `Resource` 保存粒子配置，实现一行代码触发、可复用、可预设的粒子爆发效果。

## 设计目标

- 从任意实体调用一行代码即可发射粒子。
- 配置（数量、颜色、纹理）保存为**资源预设**。
- 不同对象（木箱、敌人、可破坏墙）分配不同预设即可复用。

## 1. 命中粒子设置资源

创建 `HitParticleSettings.gd`，继承 `Resource`：

```gdscript
class_name HitParticleSettings
extends Resource

@export var count: int = 8
@export var color: Color = Color.WHITE
@export var texture: Texture2D
```

- 资源可以保存为 `.tres` 文件，作为导出变量直接分配。

## 2. 命中粒子场景

- 根节点为 `GPUParticles2D`。
- 新建 `ParticleProcessMaterial`：
  - **Emission Shape**：`Box`，尺寸约 `8×8`（让粒子从区域而不是单点发射）。
  - **Spread**：`90`。
  - **Initial Velocity**：最小 `75`，最大 `150`。
  - **Gravity**：适当增大，如默认两倍。
  - **One Shot**：开启。
  - **Explosiveness**：`0.9`（几乎同时发射，但有少量错落）。
  - **Randomness**：`1.0`。
  - **Color / Alpha Curve**：添加 `CurveTexture`，让粒子在生命末期淡出。
- 为场景附加 `HitParticles.gd`：

```gdscript
extends GPUParticles2D
class_name HitParticles

func start(direction: Vector2, settings: HitParticleSettings) -> void:
    if settings != null:
        amount = settings.count
        modulate = settings.color
        texture = settings.texture
    process_material.direction = Vector3(direction.x, direction.y, 0.0)
    emitting = true
    await finished
    queue_free()
```

- 设置 `z_index = 2`，确保可见。

## 3. 在视觉特效工厂中调用

```gdscript
const HIT_PARTICLES := preload("res://.../hit_particles.tscn")

func spawn_hit_particles(pos: Vector2, direction: Vector2, settings: HitParticleSettings) -> void:
    var particles: HitParticles = HIT_PARTICLES.instantiate()
    add_child(particles)
    particles.global_position = pos
    particles.start(direction, settings)
```

- 调用示例：`VisualEffects.spawn_hit_particles(global_position, Vector2.RIGHT, hit_settings)`。

## 4. 创建资源预设

- 在项目中新建 `Hit Particle Settings` 资源：
  - `hit_wood_large`
  - `hit_wood_medium`
  - `hit_wood_small`
  - `hit_magic_blue`
- 调整每个预设的 `count`、`color`、`texture`（从可破坏道具精灵图集裁剪木块）。
- 修改资源后，所有引用该资源的节点会自动更新。

## 5. 测试

- 在 `PlayerHUD` 添加临时按钮：
  - 获取玩家位置。
  - 指定方向（如 `Vector2.RIGHT`）。
  - 传入一个 `HitParticleSettings` 预设。
- 可快速切换不同预设，观察粒子数量、颜色、纹理、方向的变化。

## 后续应用

- 下一集将把命中粒子与**攻击区域 / 伤害区域**结合，用于敌人、可破坏物体受击反馈。

## 性能提示

- 利用 `await finished` 自动 `queue_free()`，避免残留节点。
- 若同屏粒子极多，可考虑对象池；本项目规模无需过早优化。
- 更耗性能的领域通常是大量 `AudioStreamPlayer2D` 实例，而非粒子。
