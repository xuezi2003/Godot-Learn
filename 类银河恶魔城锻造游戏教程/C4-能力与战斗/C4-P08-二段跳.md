# C4-P08：二段跳

> 本集在现有跳跃/下落状态中加入二段跳能力，通过统一的跳跃计数器管理多段跳，并确保该能力变量能被存档系统持久化。

## 1. 核心设计

- 不创建新状态，而是把二段跳逻辑融入 `JumpState` 与 `FallState`。
- 用 `jump_count` 记录当前已用跳跃次数。
- 用 `double_jump` 布尔值控制是否拥有二段跳能力。
- 从平台边缘自然滑落也算“一次跳跃”，防止无限空中连跳。

## 2. 玩家脚本更新

```gdscript
@export var double_jump: bool = false
var jump_count: int = 0
```

- `double_jump` 已在早期为存档系统预留。

## 3. 音效准备

- 在玩家节点下新建 `AudioStreamPlayer2D`：
  - `JumpAudio`
  - `LandAudio`
- 设置总线为 `SFX`。
- 在 `JumpState.enter()` 中播放 `JumpAudio`。
- 在 `FallState` 落地检测处播放 `LandAudio`。

## 4. `JumpState` 重构

### 提取 `_do_jump()`

```gdscript
func _do_jump() -> void:
    if player.jump_count > 0 and not player.double_jump:
        return
    if player.jump_count > 1:
        return

    player.velocity.y = player.jump_velocity
    player.jump_count += 1
    # 尘埃/音效已在 enter 或落地处处理
```

- 首次起跳：`jump_count == 0`，允许。
- 第二次起跳：必须 `double_jump == true` 且 `jump_count == 1`。
- 若要做三段/四段跳，只需把上限 `1` 改为更大值。

### `enter`

- 播放跳跃动画、跳跃音效、跳跃尘埃。
- 调用 `_do_jump()`。

## 5. `FallState` 处理

### `enter`

- 如果从 `CrouchState` 进入（跳下单向平台），设置 `jump_count = 1`。
- 如果从平台边缘自然滑落且 `jump_count == 0`，设置 `jump_count = 1`。
  - 这样玩家滑落时只剩一次跳跃机会，无法做两次二段跳。

### 输入处理

```gdscript
if event.is_action_pressed("jump"):
    if coyote_timer > 0.0:
        player.jump_count = 0
        return jump_state
    elif player.jump_count <= 1 and player.double_jump:
        return jump_state
```

- 土狼时间内跳跃重置计数，保证地面起跳手感。
- 空中满足条件则返回 `JumpState` 进行二次跳跃。

### 落地重置

- 当玩家落地并进入 `IdleState` 或 `RunState` 时，在 `enter` 中设置 `player.jump_count = 0`。
- 也可在 `FallState` 的落地分支中重置。

## 6. 二段跳尘埃

- 在 `JumpState.enter()` 中判断：
  - 若玩家在地面，播放普通跳跃尘埃。
  - 若不在地面（即二段跳），播放受击尘埃或自定义空中尘埃，增强视觉反馈。

## 7. 存档与加载

- `double_jump` 是导出变量，存档管理器已能保存/读取。
- 测试流程：
  1. 默认无二段跳 → 无法二段跳。
  2. 开启 `double_jump = true` → 可以二段跳。
  3. 保存游戏 → 再次加载后仍保留二段跳。

## 8. 常见边界情况

- **从边缘滑落再跳两次**：通过进入 `FallState` 时强制 `jump_count = 1` 解决。
- **土狼时间后二段跳异常**：确保土狼跳跃重置 `jump_count`。
- **跳跃键释放后漂浮**：已在 P06/P05 中修复。
