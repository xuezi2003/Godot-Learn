# C4-P04：响应式玩家攻击状态

> 本集实现一个流畅、可连招、可在多数状态下使用的玩家攻击状态，并调整手感使其接近《空洞骑士》式的移动攻击体验。

## 1. 前置回顾：受击尘埃作业

- 在攻击区域脚本 `AttackArea.gd` 的 `on_body_entered`（或对应伤害判定）中：
  - 计算命中位置：使用攻击区域自身的 `global_position.y`，但 `x` 取被击中物体的 `global_position.x`，使尘埃出现在目标中心。
  - 调用 `VisualEffects.spawn_hit_dust(pos)`。

## 2. 资源准备

- 导入“武器拖影”精灵表。
- 在玩家场景下新增 `Sprite2D` 子节点 `AttackSprite2D`，用于显示挥砍轨迹。

## 3. 攻击动画

### 玩家攻击动画

- 新建动画 `attack`，时长约 `0.3` 秒，帧间隔 `0.05` 秒。
- 玩家精灵使用剑挥出帧（如第 31–33 帧）。
- 动画保持简短，优先保证“手感”而非华丽。

### 武器拖影动画

- 为 `AttackSprite2D` 添加帧轨道：
  - 8 帧拖影精灵，沿挥砍方向播放。
- 通过**可见性关键帧**控制层级：
  - 初始帧在玩家前方。
  - 中段切换到玩家后方（使用 `z_index` 或 `show_behind_parent`）。

### 两套攻击动画

- 复制 `attack` 为 `attack2`。
- `attack2` 使用另一组挥砍帧（如第 34–36 帧），并反向调整层级。
- 状态机只使用**一个攻击状态**，通过变量决定播放哪段动画。

## 4. 创建攻击状态

### `AttackState.gd` 关键变量

```gdscript
class_name PlayerStateAttack
extends PlayerState

@export var combo_window: float = 0.2
@export var attack_speed: float = 150.0

var timer: float = 0.0
var combo: int = 0

@onready var attack_area: AttackArea = %AttackArea
@onready var attack_sprite: Sprite2D = %AttackSprite2D
@onready var audio_attack: AudioStream = preload("res://.../attack.wav")
```

### `enter`

- 隐藏 `AttackSprite2D` 初始化时。
- 连接 `AnimationPlayer.animation_finished`。
- 调用 `_do_attack()`。

### `_do_attack()`

- 根据 `combo` 选择动画名：
  - `combo == 0` → `attack`
  - `combo > 0` → `attack2`
- 播放动画。
- 激活攻击区域：`attack_area.activate()`。
- 播放攻击音效：`Audio.play_spatial_sound(audio_attack, player.global_position)`。

### 连招逻辑

- 在 `_input` 中检测攻击键：
  - 若 `timer > 0`，说明在连招窗口内，设置 `combo = wrapi(combo + 1, 0, 2)`，并再次调用 `_do_attack()`。
  - 否则按正常流程处理其他状态切换。
- 在 `_process` 中：`timer -= delta`。
- 动画完成时 `end_attack()`：
  - 若 `timer > 0` 且已按攻击，则继续连招。
  - 否则 `next_state = idle_state`（或 `fall_state`）。

### `exit`

- 断开 `animation_finished` 信号。
- 隐藏 `AttackSprite2D`。
- 重置 `timer` 与 `combo`。

## 5. 从其他状态连接攻击

在以下状态的输入处理中加入攻击检测（放在跳跃检测之前）：

- `Idle`
- `Run`
- `Jump`
- `Fall`
- `Crouch`

```gdscript
if event.is_action_pressed("attack"):
    return attack_state
```

- 这样玩家可以在奔跑、跳跃、下落、蹲伏时随时攻击。

## 6. 方向翻转与偏移

- 在玩家脚本 `update_direction()` 中：
  - 同步设置 `AttackSprite2D.flip_h = player_sprite.flip_h`。
  - 根据朝向调整 `AttackSprite2D.position.x`：
    - 朝右：`24`
    - 朝左：`-24`

## 7. 手感优化

### 攻击时移动

- 在攻击状态的 `_physics_process` 中：

```gdscript
player.velocity.x = player.direction.x * attack_speed
```

- 这样玩家可以在攻击中行走/停止，而不是被锁定。

### 保留可变跳跃高度

- 将“松开跳跃键时 Y 速度减半”的逻辑从 `JumpState` 移到**玩家脚本**的 `_unhandled_input` 中。
- 确保攻击期间松开跳跃键仍能正常降低跳跃高度。

## 8. 结果

- 玩家可以在几乎所有动作中发动攻击。
- 两次攻击交替播放，形成基础连招。
- 攻击有音效、轨迹、命中尘埃，手感流畅。
- 下一集将基于这些系统制作可破坏物体。
