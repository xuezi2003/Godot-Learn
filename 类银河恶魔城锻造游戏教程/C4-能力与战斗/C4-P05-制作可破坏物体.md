# C4-P05：制作可破坏物体

> 本集利用已有的攻击区域、伤害区域、视觉特效与音频系统，建造测试假人、通用 `Breakable` 节点以及可破坏墙壁，让战斗与世界互动更具反馈感。

## 1. 开场代码修正

在 P04 实现攻击后，出现了两个跳跃相关的小 bug：

### 1. 跳跃 + 攻击导致的“意外二段跳”

- **原因**：攻击结束后进入 `Idle`，再进入 `Fall` 时触发土狼时间，允许再次跳跃。
- **修复**：
  - 在 `AttackState.end_attack()` 中不直接返回 `Idle`，而是判断：
    -  grounded → `Idle`
    -  airborne → `Fall`
  - 在 `FallState.enter()` 中加入条件：若 `previous_state == attack_state`，不启动土狼计时器。

### 2. 松开跳跃键后漂浮感

- **原因**：跳跃速度减半逻辑被移到玩家脚本后缺少方向/速度条件。
- **修复**：仅在松开跳跃键且 Y 速度向上（`velocity.y < 0`）时，才将速度减半。

## 2. 测试假人

### 场景结构

- `TestDummy`（`Node2D`）
  - `BaseSprite2D`：底座。
  - `BodySprite2D`：可被击打的躯干；设置 `offset.y = -19`，使旋转中心位于底部。
  - `DamageArea`（伤害区域）+ `CollisionShape2D`（矩形）。

### 脚本要点

```gdscript
extends Node2D

@export var particles: Array[HitParticleSettings]
@export var wobble_angle: float = 4.0
@export var wobble_speed: float = 0.1

var wobble_count: int = 0
var tween: Tween

@onready var body_sprite: Sprite2D = %BodySprite2D
@onready var damage_area: DamageArea = %DamageArea

func _ready() -> void:
    damage_area.damage_taken.connect(_on_damage_taken)

func _on_damage_taken(attack_area: AttackArea) -> void:
    var direction := 1.0 if attack_area.global_position.x > global_position.x else -1.0
    var pos := global_position + Vector2(0, -30)
    for p in particles:
        VisualEffects.spawn_hit_particles(pos, Vector2(direction, 0.0), p)
    _start_wobble(direction)

func _start_wobble(direction: float) -> void:
    if tween != null:
        tween.kill()
    wobble_count = 5
    _wobble(direction)

func _wobble(direction: float) -> void:
    if wobble_count <= 0:
        return
    tween = create_tween()
    tween.tween_property(body_sprite, "rotation_degrees",
        wobble_angle * wobble_count * direction, wobble_speed * 0.05)
    await tween.finished
    wobble_count -= 1
    _wobble(-direction)
```

- 在 `DamageArea` 上分配通用打击音效。
- **确保 `DamageArea` 的碰撞层/遮罩能被玩家的 `AttackArea` 检测到**（如放在可破坏层或敌人层）。
- 通过导出变量可在编辑器中快速调整摇摆幅度和速度。

## 3. 通用 `Breakable` 节点

### 节点设计

- 新建脚本 `Breakable.gd`，继承 `Node2D`（工具脚本，带图标）。
- 信号：`destroyed`。
- 导出变量：
  - `health: float = 3.0`
  - `fixed_hit_count: bool = false`
  - `hit_particles: Array[HitParticleSettings]`
  - `destroy_particles: Array[HitParticleSettings]`
  - `hit_audio: AudioStream`
  - `destroy_audio: AudioStream`
  - `emit_offset: Vector2`

### 配置警告

```gdscript
func _get_configuration_warnings() -> PackedStringArray:
    if not _has_damage_area():
        return ["需要一个 DamageArea 子节点才能工作"]
    return []
```

- 在编辑器中忘记添加 `DamageArea` 时会显示警告。

### 受伤与销毁逻辑

```gdscript
func _on_damage_taken(attack_area: AttackArea) -> void:
    if fixed_hit_count:
        health -= 1
    else:
        health -= attack_area.damage

    var direction := sign(global_position.x - attack_area.global_position.x)
    var pos := global_position + emit_offset

    if health > 0:
        Audio.play_spatial_sound(hit_audio, pos)
        for p in hit_particles:
            VisualEffects.spawn_hit_particles(pos, Vector2(direction, 0.0), p)
    else:
        _destroy(pos, direction)

func _destroy(pos: Vector2, direction: float) -> void:
    _clear_collisions()
    Audio.play_spatial_sound(destroy_audio, pos)
    for p in destroy_particles:
        VisualEffects.spawn_hit_particles(pos, Vector2(direction, 0.0), p)
    var tween := create_tween()
    tween.tween_property(self, "modulate:a", 0.0, 0.4)
    await tween.finished
    queue_free()

func _clear_collisions() -> void:
    for child in get_children():
        if child is StaticBody2D:
            child.queue_free()
```

- 命中与销毁使用不同音频和粒子组合。
- 销毁时先禁用静态碰撞，再做淡出，避免玩家卡在碰撞上。

### 补充 `damage_taken` 信号

- 在 `Breakable.gd` 的受伤处理中加入 `damage_taken.emit()`，使外部节点（如下集的能力宝珠）能监听每次受击。

## 4. 可破坏木箱

1. 在 `Breakable` 节点下添加 `Sprite2D`、圆形碰撞形状、`DamageArea`。
2. 设置碰撞层为可破坏层。
3. 分配命中/销毁粒子与音频。
4. 启用 `fixed_hit_count` 可让它固定受击次数。
5. 右键点击根节点 → **保存分支为场景**，命名为 `crate_wood.tscn`。
6. 给木箱添加 `StaticBody2D` + 碰撞形状，使其可站立/阻挡。
7. 调整精灵 `Centered` 与碰撞对齐，避免奇数像素造成的偏移。

## 5. 可破坏墙壁

- 在瓦片地图层上绘制普通墙壁。
- 在墙壁位置实例化一个 `Breakable` 节点，并添加 `DamageArea` 覆盖墙体区域。
- 使用破损墙壁精灵作为可视化。
- 设置 `fixed_hit_count = true`（如 2 次）。
- 销毁时 `queue_free()` 移除自身碰撞，玩家即可穿过。

## 6. 结果

- 测试假人：受击摇摆、粒子、音效。
- 通用 `Breakable`：任何带伤害区域的物体都能被赋予生命值、命中反馈、销毁反馈。
- 可破坏木箱与墙壁：展示道具与隐藏区域的构建方式。
