# C4-P12：能力拾取

> 本集把能力系统收尾：创建可被攻击摧毁的能力宝珠，摧毁后奖励玩家对应能力、持久化到存档，并在暂停菜单中显示已获得的能力图标。

## 1. 核心设计

- 使用 `Breakable` 节点作为基础，让玩家通过攻击击碎宝珠。
- 每种能力一个可拾取节点，分配不同图标动画。
- 摧毁后设置玩家脚本中的能力布尔值，并通过 `SaveManager` 持久化。
- 再次加载游戏时，已获取的能力宝珠不再出现。

## 2. 场景结构：`AbilityPickup`

- `AbilityPickup`（`Node2D`，工具脚本，带图标）
  - `AbilitySprite2D`：显示能力动态图标（冲刺、二段跳、地面猛击、变形）。
  - `OrbSprite2D`：玻璃球外壳，根据受击次数切换帧显示裂痕。
  - `OrbAnimationPlayer`：玻璃球破碎 + 淡出动画。
  - `AbilityAnimationPlayer`：能力图标循环动画。
  - `Breakable` 节点
    - `DamageArea` + `CollisionShape2D`（圆形）
    - 命中/销毁音频与粒子

## 3. 资源准备

- 导入 `ability_sprite.png` 精灵表（5×5）：
  - 第 0 行：冲刺
  - 第 1 行：二段跳
  - 第 2 行：地面猛击
  - 第 3 行：变形翻滚
  - 第 4 行：玻璃球受击/破碎帧
- 导入能力命中音效与获取音效。

## 4. `AbilityPickup.gd` 脚本

### 类型枚举

```gdscript
@tool
extends Node2D
class_name AbilityPickup

enum Type { DOUBLE_JUMP, DASH, GROUND_POUND, MORPH }

@export var type: Type = Type.DOUBLE_JUMP:
    set(value):
        type = value
        _set_animation()
```

### 获取能力名称

```gdscript
func _get_ability_name() -> String:
    match type:
        Type.DOUBLE_JUMP: return "double_jump"
        Type.DASH:        return "dash"
        Type.GROUND_POUND: return "ground_pound"
        Type.MORPH:       return "morph_roll"
    return ""
```

### 设置能力图标动画

```gdscript
func _set_animation() -> void:
    if ability_animation_player == null:
        ability_animation_player = %AbilityAnimationPlayer
    ability_animation_player.play(_get_ability_name())
```

### `_ready`

```gdscript
func _ready() -> void:
    if Engine.is_editor_hint():
        return

    _set_animation()

    if SaveManager.persistent_data.get_or_add(_get_ability_name(), "") == "obtained":
        queue_free()
        return

    breakable.damage_taken.connect(_on_damage_taken)
    breakable.destroyed.connect(_on_destroyed)
```

### 受击更新玻璃球裂痕

```gdscript
func _on_damage_taken(_attack_area: AttackArea) -> void:
    orb_sprite.frame += 1
```

### 摧毁时奖励能力

```gdscript
func _on_destroyed() -> void:
    _reward_ability()
    SaveManager.persistent_data[_get_ability_name()] = "obtained"
    SaveManager.save_game()
    await orb_animation_player.animation_finished
    queue_free()

func _reward_ability() -> void:
    var player = get_tree().get_first_node_in_group("player")
    if player == null:
        return
    match type:
        Type.DOUBLE_JUMP: player.double_jump = true
        Type.DASH:        player.dash = true
        Type.GROUND_POUND: player.ground_pound = true
        Type.MORPH:       player.morph = true
```

## 5. `Breakable.gd` 补充信号

- 在 P05 的基础上新增 `damage_taken` 信号。
- 在 `Breakable._on_damage_taken` 中每次受伤时 `emit_signal("damage_taken", attack_area)`。
- 生命值归零时 `emit_signal("destroyed")`。

## 6. 玻璃球动画

- `default`：完整玻璃球（第 20 帧）。
- `destroy`：从第 22 帧开始，持续约 0.2 秒，最后设置 `OrbSprite2D.visible = false`。
- 同时为 `AbilitySprite2D` 做透明度淡出补间，使图标消失更自然。

## 7. 持久化

- 使用 `SaveManager.persistent_data` 存储键值：
  - 键：`"double_jump"`、`"dash"`、`"ground_pound"`、`"morph_roll"`
  - 值：`"obtained"`
- 读取时若已获取，直接 `queue_free()`，避免重复出现。

## 8. 暂停菜单能力列表

### UI

- 在 `PauseMenu` 的“能力”区域添加 `VBoxContainer`（命名为 `AbilityList`）。
- 添加 4 个 `TextureRect`：
  - `Ability_DoubleJump`
  - `Ability_Dash`
  - `Ability_GroundPound`
  - `Ability_MorphRoll`
- 使用能力精灵图集裁剪 32×32 区域作为图标。

### 脚本：`AbilityList.gd`

```gdscript
extends VBoxContainer

@onready var ability_double_jump: TextureRect = %Ability_DoubleJump
@onready var ability_dash: TextureRect = %Ability_Dash
@onready var ability_ground_pound: TextureRect = %Ability_GroundPound
@onready var ability_morph_roll: TextureRect = %Ability_MorphRoll

func _ready() -> void:
    var player = get_tree().get_first_node_in_group("player")
    if player == null:
        return
    ability_double_jump.visible = player.double_jump
    ability_dash.visible = player.dash
    ability_ground_pound.visible = player.ground_pound
    ability_morph_roll.visible = player.morph
```

- 暂停时只显示已拥有的能力图标。

## 9. 测试流程

1. 玩家没有二段跳 → 无法二段跳。
2. 击碎“二段跳”宝珠 → 获得二段跳。
3. 暂停 → 看到对应图标亮起。
4. 保存并退出。
5. 重新加载 → 宝珠已消失，能力仍然保留。
