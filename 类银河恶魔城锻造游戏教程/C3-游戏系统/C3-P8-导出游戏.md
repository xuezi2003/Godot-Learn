# C3-P8：导出游戏

> 本集讲解如何将 Godot 项目导出为 Windows 桌面版与网页版，区分调试版与发布版，并介绍如何使用 `OS.is_debug_build()` 锁定调试代码。

## 1. 导出前的项目设置

### 1.1 设置主场景

- 进入 **项目设置 → 运行 → 主场景**。
- 将主场景改为标题界面 `TitleScreen.tscn`，确保玩家启动游戏时首先看到标题画面。

### 1.2 项目信息

- **项目设置 → 应用 → 配置**：
  - 名称：确保显示正确的游戏名称。
  - 公司/产品名称：填写开发者与游戏名。
- 建议设置自定义图标：
  - Windows 原生图标（`.ico`）。
  - macOS 原生图标（`.icns`）。
  - 默认 Godot 图标不够专业，建议替换。

### 1.3 启动闪屏

- **项目设置 → 应用 → 启动闪屏**：
  - 可设置背景颜色或图片，替换 Godot 默认启动画面。
  - 也可设置为全屏显示。

## 2. 导出 Windows 桌面版

### 2.1 添加导出预设

- 打开 **项目 → 导出**。
- 点击 **添加**，选择 **Windows 桌面**。

### 2.2 安装导出模板

- 如果提示缺少导出模板，点击 **管理导出模板**。
- 选择对应 Godot 版本，点击 **下载并安装**。
- 安装完成后返回导出窗口，警告应消失。

### 2.3 导出配置

常见配置项：

| 选项 | 说明 |
|---|---|
| 架构 | 默认 x86_64，适合大多数现代 PC |
| 可执行文件名称 | 生成的 `.exe` 文件名 |
| 应用程序图标 | Windows `.ico` |
| 版本信息 | 公司名、产品名、文件版本等 |
| 功能/资源补丁 | 高级选项，可添加功能标志 |
| 加密 | 发布时通常需要启用 |

### 2.4 调试版导出

- 勾选 **使用调试导出**。
- 导出后会生成两个可执行文件：
  - `项目名.exe`：主程序。
  - `项目名.console.exe`：带控制台窗口，可查看打印语句与错误日志。

### 2.5 发布版导出

- 取消勾选 **使用调试导出**。
- 只生成主程序，无控制台窗口。
- 发布版编译优化更好，性能更优，适合最终发布。

## 3. 调试代码锁定

### 3.1 问题

- 玩家脚本和保存管理器中包含 F5/F7 调试快捷键、Shift+加号/减号调整生命值等测试代码。
- 这些代码不应出现在发布版中。

### 3.2 使用 `OS.is_debug_build()`

```gdscript
func _unhandled_input(event: InputEvent) -> void:
    if not OS.is_debug_build():
        return
    
    # 仅调试版可用的快捷键
    if event is InputEventKey and event.pressed:
        if event.keycode == KEY_F5:
            SaveManager.save_game()
        elif event.keycode == KEY_F7:
            SaveManager.load_game(SaveManager.current_slot)
```

- `OS.is_debug_build()` 在以下情况返回 `true`：
  - 在编辑器中运行。
  - 使用调试导出模板导出的构建。
- 在发布版中返回 `false`。

### 3.3 应用位置

- 玩家脚本中的生命值调试快捷键。
- 保存管理器中的 F5/F7 保存加载快捷键。
- 任何仅用于测试的代码。

## 4. 导出网页版

### 4.1 添加导出预设

- 在导出窗口点击 **添加**，选择 **Web**。

### 4.2 重要设置

| 选项 | 建议 |
|---|---|
| 线程支持 | 默认关闭，兼容性更好；开启可能提升性能但部分浏览器不支持 |
| 纹理压缩 | 移动端目标平台可考虑启用 |
| 自定义图标/颜色 | 可配置网页加载时的图标与背景色 |

### 4.3 导出文件

- 将网页版导出到一个文件夹。
- 关键文件是 `index.html`，上传至 itch.io 等网页平台时需要。
- 将文件夹压缩为 `.zip`。

### 4.4 上传到 itch.io

1. 在 itch.io 创建项目，选择 **HTML** 类型。
2. 上传包含 `index.html` 的 `.zip` 文件。
3. 勾选 **This file will be played in the browser**。
4. 保存后即可在网页中运行游戏。

## 5. 网页版性能问题

### 5.1 已知问题

- Godot 4 的网页支持相比 Godot 3 在某些方面有所倒退，复杂项目可能遇到卡顿。
- 示例项目中从第一关进入第二关时会出现明显卡顿。

### 5.2 排查与优化

- 禁用或替换 GPU 粒子为 CPU 粒子。
- 减少复杂着色器与后期处理。
- 在网页构建中禁用部分特效，可通过 `OS.has_feature("web")` 做条件判断。

```gdscript
if OS.has_feature("web"):
    gpu_particles.emitting = false
```

### 5.3 平台检测注意

- 在移动设备浏览器中运行时，`OS.has_feature("web")` 返回 `true`，`OS.has_feature("mobile")` 返回 `false`。
- 只有在导出为原生 APK 并运行时，才会被识别为移动版。

## 6. 测试与迭代

- 每次达到里程碑都为 PC、网页或主要目标平台导出构建。
- 发送给朋友或测试用户，收集反馈。
- 检查性能、存档、音频、UI 焦点等是否正常工作。

## 7. 发布建议

- 正式发布前启用资源加密。
- 为各平台准备正确的图标与版本信息。
- 使用发布版导出，确保调试代码被锁定。
- 在目标平台上充分测试，尤其是网页版需要多浏览器验证。
