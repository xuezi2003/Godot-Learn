# C3-P9：音频播放器池化【优化篇】

> 本集将频繁创建和销毁的 2D 音效播放器改造为对象池，以减少节点创建销毁开销，并通过性能分析展示池化带来的收益。

## 1. 池化系统概念

### 1.1 为什么需要池化

- 当前 `play_spatial_sound()` 每次播放音效都会 `new` 一个 `AudioStreamPlayer2D`，播放结束后 `queue_free()`。
- Godot 处理节点创建/销毁效率较高，但音效密集时仍会产生额外开销。
- 对象池通过预创建一组节点并循环复用，避免频繁的内存分配与释放。

### 1.2 池化的 trade-off

| 优点 | 缺点 |
|---|---|
| 减少节点创建/销毁开销 | 池大小固定，若同时播放音效超过池大小会切断旧音效 |
| 稳定内存占用 | 较长的音效（如背景音乐）不适合放入池中 |
| 适合短促频繁的音效 | 需要为不想被切断的音效提供旁路 |

## 2. 音频池实现

### 2.1 新增变量

```gdscript
const POOL_SIZE := 32

var audio_pool: Array[AudioStreamPlayer2D] = []
var audio_index: int = 0
```

### 2.2 初始化对象池

```gdscript
func _ready() -> void:
    # ... UI 音频初始化 ...
    
    for i in POOL_SIZE:
        var player := AudioStreamPlayer2D.new()
        add_child(player)
        player.bus = "SFX"
        audio_pool.append(player)
```

- 在游戏启动时创建 32 个 `AudioStreamPlayer2D` 节点，一直存在直到游戏结束。

### 2.3 修改 `play_spatial_sound`

```gdscript
func play_spatial_sound(stream: AudioStream, pos: Vector2, ignore_pool: bool = false) -> void:
    if ignore_pool:
        # 重要音效不走池化，直接创建独立播放器
        var player := AudioStreamPlayer2D.new()
        add_child(player)
        player.bus = "SFX"
        player.global_position = pos
        player.stream = stream
        player.finished.connect(player.queue_free)
        player.play()
        return
    
    var player := audio_pool[audio_index]
    player.global_position = pos
    player.stream = stream
    player.play()
    
    audio_index = wrapi(audio_index + 1, 0, POOL_SIZE)
```

### 2.4 关键说明

- `ignore_pool = false` 时默认使用池化。
- `ignore_pool = true` 时创建独立播放器，确保重要音效不会被池中的其他音效覆盖。
- `wrapi()` 使索引在 0~31 之间循环。

## 3. 不放入池中的音效

- 持续较长的音效。
- 非常关键、不能被切断的音效。
- 调用时显式传入 `ignore_pool = true`：

```gdscript
Audio.play_spatial_sound(important_sound, global_position, true)
```

## 4. 性能分析

### 4.1 测试场景

- 在测试场景中添加脚本，每 0.01 秒播放一次音效。
- 通过按键切换是否使用池化。

### 4.2 使用 Godot Profiler

- 打开 **调试器 → 性能分析器（Profiler）**。
- 启用 **自动启动**。
- 运行测试场景并大量播放音效。
- 关注 **Audio Thread** 的帧时间。

### 4.3 结果对比

| 模式 | Audio Thread 峰值 |
|---|---|
| 使用池化 | 约 0.3~0.6 ms |
| 不使用池化 | 可达 1.5~2.25 ms |

- 峰值约降低 3~4 倍，具体取决于音频文件大小、时长和比特率。
- 虽然对小型项目影响不明显，但音效密集的游戏中收益显著。

## 5. 对象池的通用应用

- 对象池不仅适用于音频，还常用于：
  - 子弹/弹幕系统。
  - 粒子效果。
  - 频繁生成销毁的敌人或道具。
- 核心思想一致：预分配对象、循环复用、避免频繁创建销毁。

## 6. 测试要点

- 普通音效通过池化正常播放。
- 同时触发大量音效时无异常中断或崩溃。
- 重要音效使用 `ignore_pool = true` 时不会被其他音效覆盖。
- 远程场景树中应只看到固定数量的 `AudioStreamPlayer2D` 节点，不会动态新增。

## 7. 总结

- 音频池化是一种简单有效的优化手段。
- 32 个池化播放器足以满足大多数银河恶魔城游戏的音效需求。
- 根据项目规模和音效密度调整 `POOL_SIZE`。
- 对不能切断的重要音效提供旁路机制。
