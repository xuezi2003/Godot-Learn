# C3-P5：地图和暂停菜单 - 2

> 本集在暂停菜单中实现完整的地图系统，包括可复用的地图节点场景、自动从关卡场景读取尺寸与出口位置、探索状态持久化以及玩家位置指示器。

## 1. 地图节点场景

### 1.1 创建地图节点

- 在 `PauseMenu/` 中创建 `MapNode.tscn`。
- 根节点类型：**Control**，命名为 `MapNode`。
- 设置 `Custom Minimum Size`：宽度 12，高度 6（基于 480×270 关卡尺寸除以 40 的缩放因子）。
- 保存并打开 `MapNode.tscn`。

### 1.2 节点结构

```
MapNode (Control)
├── TopBorder (ColorRect)      # 上边框
├── BottomBorder (ColorRect)   # 下边框
├── LeftBorder (ColorRect)     # 左边框
├── RightBorder (ColorRect)    # 右边框
├── LevelNameLabel (Label)     # 编辑器中显示关卡名称
└── TransitionsContainer (Control)  # 出口/过渡小块容器
```

- 四个 `ColorRect` 作为 1 像素宽的边框。
- `LevelNameLabel` 仅在编辑器中显示，游戏中隐藏。
- `TransitionsContainer` 用于存放表示关卡连接的小黑块。

### 1.3 图标与工具脚本

- 为 `MapNode` 附加脚本 `map_node.gd`。
- 添加 `class_name MapNode`，`@tool`（工具脚本），以及自定义图标。
- 重新加载项目，使场景树中显示地图节点图标。

## 2. 地图节点脚本

### 2.1 导出变量

```gdscript
@tool
class_name MapNode
extends Control

const SCALE_FACTOR := 40.0

@export_file("*.tscn") var linked_scene: String:
    set(value):
        if linked_scene != value:
            linked_scene = value
            if Engine.is_editor_hint():
                update_node()

@export var entrances_top: Array[float] = []
@export var entrances_right: Array[float] = []
@export var entrances_bottom: Array[float] = []
@export var entrances_left: Array[float] = []

var indicator_offset := Vector2.ZERO

@export_tool_button("Update", "EditorIcons") var update_node_action := update_node
```

- `linked_scene`：关联的关卡场景文件。
- `entrances_*`：四个方向出口在地图节点内的偏移量。
- `indicator_offset`：玩家位置指示器相对于地图节点左上角的偏移。
- `@export_tool_button`：在编辑器 Inspector 中显示“Update”按钮，手动刷新节点。

### 2.2 更新节点

```gdscript
func update_node() -> void:
    if linked_scene.is_empty():
        return
    # 加载关卡场景
    var packed_scene := load(linked_scene) as PackedScene
    var instance := packed_scene.instantiate()
    
    # 查找 LevelBounds 节点
    var bounds := instance.get_node("LevelBounds") as Control
    if bounds == null:
        instance.queue_free()
        return
    
    # 设置地图节点尺寸
    var scaled_size := bounds.size / SCALE_FACTOR
    custom_minimum_size = scaled_size
    size = scaled_size
    
    # 记录边界偏移
    indicator_offset = bounds.position
    
    # 收集所有 Transition 节点
    var transitions := instance.find_children("*", "Area2D", true)
    # 根据过渡位置计算 entrances_* 并创建过渡块 ...
    
    instance.queue_free()
```

### 2.3 创建过渡块

- 遍历关卡中的所有过渡区域。
- 根据过渡区域相对于 `LevelBounds` 的位置，计算在地图节点内应显示的小黑块位置。
- 在 `TransitionsContainer` 中动态添加 `ColorRect` 表示出口。

### 2.4 显示/隐藏探索状态

```gdscript
func _ready() -> void:
    if Engine.is_editor_hint():
        return
    
    # 游戏中移除编辑器标签
    $LevelNameLabel.queue_free()
    
    # 创建过渡块
    create_transition_blocks()
    
    # 根据是否已探索决定是否显示
    if SaveManager.is_area_discovered(linked_scene):
        visible = true
    else:
        visible = false
```

- 需要在 `SaveManager` 中维护 `discovered_areas` 数组，并提供查询方法。

## 3. 在暂停菜单中布置地图

### 3.1 拖放地图节点

- 在 `PauseMenu → PauseScreen → Map` 中实例化多个 `MapNode`。
- 每个 `MapNode` 关联一个关卡场景。
- 点击 Inspector 中的 **Update** 按钮，自动调整尺寸并绘制出口。

### 3.2 区域颜色

- 为不同区域设置不同颜色：
  - 森林区域：黄绿色。
  - 保存房间：粉色。
  - 地牢区域：蓝灰色。
- 使用 Godot 调色板保存常用颜色，便于复用。

## 4. 探索状态记录

### 4.1 进入区域时记录

- 在关卡加载或玩家进入关卡边界时，调用：

```gdscript
SaveManager.discover_area(current_scene_path)
```

- `discover_area` 将场景 UID 或路径加入 `discovered_areas` 数组。

### 4.2 保存与加载

- `discovered_areas` 作为保存数据的一部分写入存档文件。
- 加载存档时恢复该数组。

### 4.3 查询方法

```gdscript
func is_area_discovered(scene_path: String) -> bool:
    return scene_path in discovered_areas
```

## 5. 玩家位置指示器

### 5.1 在暂停菜单中添加指示器

- 在 `Map` 控件中添加一个 `ColorRect` 或 `TextureRect` 作为玩家位置指示器。
- 命名为 `PlayerIndicator`。

### 5.2 更新指示器位置

- 当暂停菜单打开时，根据当前关卡对应的 `MapNode` 和玩家在世界中的相对位置计算指示器位置：

```gdscript
func update_player_indicator() -> void:
    var current_map_node := find_map_node_for_scene(current_scene_uid)
    if current_map_node == null:
        return
    
    var player_pos := player.global_position
    var relative_pos := player_pos - current_map_node.indicator_offset
    player_indicator.position = current_map_node.position + relative_pos / MapNode.SCALE_FACTOR
```

- 需要在暂停菜单脚本中持有玩家引用。

## 6. 总结

| 组件 | 作用 |
|---|---|
| `MapNode` | 代表一个关卡房间，自动从场景读取尺寸与出口 |
| `SaveManager.discovered_areas` | 记录已探索的区域 |
| `PlayerIndicator` | 在地图上显示玩家当前位置 |
| `Update` 工具按钮 | 手动刷新地图节点尺寸与出口布局 |
