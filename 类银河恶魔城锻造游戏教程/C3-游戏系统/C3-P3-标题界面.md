# C3-P3：标题界面

> 本集制作游戏的标题界面，包含动态 Logo 动画、新游戏与加载游戏菜单，并实现与全局保存管理器的完整对接，让玩家可以通过标题菜单开始新游戏或加载已有存档槽。

## 1. 场景搭建

### 1.1 创建标题画面文件夹与场景

- 在项目根目录下创建 `TitleScreen/` 文件夹。
- 新建场景，根节点类型选择 **CanvasLayer**。
  - `Layer = 5`，确保标题画面位于其他 UI 之上。

### 1.2 背景

- 在 `CanvasLayer` 下添加一个 `Control` 节点作为容器（命名为 `MainMenu` 或 `PauseScreenSpace` 等）。
- 将现有关卡中的瓦片地图层/装饰节点复制粘贴到标题画面作为背景，使其美观且对称。

### 1.3 主菜单结构

```
TitleScreen (CanvasLayer)
└── Control
    ├── VBoxContainer (MainMenu)
    │   ├── TextureRect/Sprite2D (Logo)
    │   ├── Button (NewGameButton)
    │   └── Button (LoadGameButton)
    ├── VBoxContainer (NewGameMenu)
    │   ├── Label
    │   └── Button×3 (NewSlot01/02/03)
    └── VBoxContainer (LoadGameMenu)
        ├── Label
        └── Button×3 (LoadSlot01/02/03)
```

- 三个菜单通过显隐切换。
- 为按钮启用 **唯一名称访问（Access as Unique Name）**，便于脚本引用。

### 1.4 默认字体设置

- 进入 **项目设置 → GUI → Theme → Custom Font**，选择 `Allegard` 字体。
- 保存并重新加载项目，之后所有新建按钮/标签都会自动使用该字体。

## 2. Logo 动画

### 2.1 导入与配置

- 将 `MetroidvaniaForgeLogo.png` 拖入 `TitleScreen/`。
- 在 `VBoxContainer` 中添加 `Control` 节点作为 Logo 容器，设置最小尺寸。
- 在容器内添加 `Sprite2D`，拖入 Logo 精灵表。
- Logo 精灵表参数：`Hframes = 8`，`Vframes = 3`。

### 2.2 动画

- 为 Logo 容器添加 `AnimationPlayer`。
- **Start 动画**（播放一次）：
  - 时长约 0.5 秒，逐帧显示剑刺入、Logo 形成的画面。
- **Loop 动画**（循环播放）：
  - 时长 1 秒，显示 Logo 悬浮、魔法液滴漂浮并爆开的效果。
- 在 Start 动画完成后自动切换到 Loop 动画。

```gdscript
func _on_animation_finished(anim_name: StringName) -> void:
    if anim_name == "Start":
        animation_player.play("Loop")
```

## 3. 标题界面脚本

### 3.1 获取引用

```gdscript
@onready var main_menu: VBoxContainer = %MainMenu
@onready var new_game_menu: VBoxContainer = %NewGameMenu
@onready var load_game_menu: VBoxContainer = %LoadGameMenu

@onready var new_game_button: Button = %NewGameButton
@onready var load_game_button: Button = %LoadGameButton

@onready var new_slot_buttons: Array[Button] = [%NewSlot01, %NewSlot02, %NewSlot03]
@onready var load_slot_buttons: Array[Button] = [%LoadSlot01, %LoadSlot02, %LoadSlot03]

@onready var animation_player: AnimationPlayer = $AnimationPlayer
```

### 3.2 菜单显示函数

```gdscript
func show_main_menu() -> void:
    main_menu.visible = true
    new_game_menu.visible = false
    load_game_menu.visible = false
    new_game_button.grab_focus()

func show_new_game_menu() -> void:
    main_menu.visible = false
    new_game_menu.visible = true
    load_game_menu.visible = false
    new_slot_buttons[0].grab_focus()

func show_load_game_menu() -> void:
    main_menu.visible = false
    new_game_menu.visible = false
    load_game_menu.visible = true
    load_slot_buttons[0].grab_focus()
```

### 3.3 动态按钮文本

- 如果某个存档槽已存在存档文件，新游戏菜单中对应按钮文本改为 `替换槽位 X`；否则显示 `开始新游戏 X`。
- 加载游戏菜单中，不存在的槽位按钮直接禁用。

```gdscript
for i in range(3):
    var exists := SaveManager.save_file_exists(i)
    new_slot_buttons[i].text = "替换槽位 %d" % (i + 1) if exists else "开始新游戏 %d" % (i + 1)
    load_slot_buttons[i].disabled = not exists
```

- 为此需要在 `SaveManager` 中扩展 `get_file_name(slot: int)` 与 `save_file_exists(slot: int)`。

### 3.4 返回主菜单

- 在未处理输入中检测 `ui_cancel`：

```gdscript
func _unhandled_input(event: InputEvent) -> void:
    if event.is_action_pressed("ui_cancel") and not main_menu.visible:
        show_main_menu()
```

- 确保在项目设置的 UI 输入映射中绑定了取消键（键盘 Esc、手柄 Circle/B 等）。

## 4. 保存管理器调整

### 4.1 让 `get_file_name` 接受槽位参数

```gdscript
func get_file_name(slot: int) -> String:
    return SAVE_PATH % slot
```

- 将内部所有 `get_file_name()` 调用替换为 `get_file_name(current_slot)`。

### 4.2 新增 `save_file_exists`

```gdscript
func save_file_exists(slot: int) -> bool:
    return FileAccess.file_exists(get_file_name(slot))
```

### 4.3 让 `create_new_save_game` 与 `load_game` 接受槽位参数

```gdscript
func create_new_save_game(slot: int) -> void:
    current_slot = slot
    # ... 创建存档 ...
    save_game()
    file.close()  # 关键：确保文件已写入磁盘
    load_game(slot)

func load_game(slot: int) -> void:
    current_slot = slot
    # ... 读取存档并触发场景过渡 ...
```

- 创建新存档后立即调用 `load_game`，需要确保文件访问已关闭，否则加载时可能读到空文件或旧数据。
- Godot 文档说明 `FileAccess` 会在变量超出作用域时自动关闭，但作者在实践中发现：创建存档后立刻加载同一文件时，若不显式调用 `file.close()`，会出现加载失败或数据未更新的 bug。
- > **建议**：在 `save_game()` 写入完成后、调用 `load_game()` 之前，显式 `file.close()`，确保数据已落盘。

## 5. 按钮连接

### 5.1 主菜单按钮

```gdscript
new_game_button.pressed.connect(show_new_game_menu)
load_game_button.pressed.connect(show_load_game_menu)
```

### 5.2 新游戏槽位按钮

```gdscript
for i in range(3):
    new_slot_buttons[i].pressed.connect(_on_new_game_pressed.bind(i))

func _on_new_game_pressed(slot: int) -> void:
    SaveManager.create_new_save_game(slot)
    SceneManager.transition_scene("关卡1的UID", "", Vector2.ZERO, 0)
```

### 5.3 加载游戏槽位按钮

```gdscript
for i in range(3):
    load_slot_buttons[i].pressed.connect(_on_load_game_pressed.bind(i))

func _on_load_game_pressed(slot: int) -> void:
    SaveManager.load_game(slot)
```

## 6. 玩家实例清理

- 从标题界面开始新游戏或加载游戏时，场景中可能残留上一个会话的玩家实例。
- 在 `create_new_save_game` 后调用 `load_game`，`load_game` 内部会重新设置玩家，避免满级玩家进入新游戏的 bug。

## 7. 测试要点

- 标题画面加载后自动播放 Logo 动画并循环。
- 键盘/手柄可以聚焦按钮并在菜单间切换。
- 新游戏菜单根据存档存在与否显示“替换”或“开始新游戏”。
- 加载游戏菜单仅启用存在存档的槽位。
- 开始新游戏或加载后正确进入对应场景。
- 存档文件在用户数据文件夹中正确生成。
