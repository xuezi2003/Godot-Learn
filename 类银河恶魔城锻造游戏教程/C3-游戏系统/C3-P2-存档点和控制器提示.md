# C3-P2：存档点和控制器提示

> 本集实现可复用的存档点场景、根据控制器类型动态变化的输入提示，并引入全局消息总线来解耦玩家与可交互对象之间的通信。

## 核心功能

1. **存档点场景**：玩家进入区域后按下交互键即可保存游戏并可选地治疗玩家。
2. **输入提示节点**：附着在玩家身上，根据当前使用的输入设备显示对应按键图标。
3. **消息总线系统**：全局自动加载脚本，用于在系统间发送松散耦合的信号。

## 1. 资源导入

- 在 `Sprites/Player/` 中导入**输入图标**素材。
- 在 `Sprites/Common/` 中导入**保存点图标**、通用图标以及 `Allegard` 字体。
- 在 `Sprites/Common/SavePoint/` 中导入保存点精灵 `SavePoint.png`。

## 2. 存档点场景搭建

### 2.1 节点结构

```
SavePoint (Node2D)
├── Sprite2D              # 保存点视觉
├── PointLight2D          # 粉色光晕
├── Area2D                # 触发交互区域
│   └── CollisionShape2D  # 矩形碰撞，仅检测玩家层
├── Label                 # "游戏已保存" 提示文字
└── AnimationPlayer       # 提示文字动画
```

### 2.2 视觉配置

- **Sprite2D**：拖入 `SavePoint.png`，调整位置使其与场景中心对齐。
- **PointLight2D**：
  - 使用 `GradientTexture2D`，尺寸 256×256，镜像填充。
  - 颜色设为粉色，亮度低于火把，营造微妙的发光效果。
- **Label**：
  - 使用 `Allegard` 字体，居中对齐，尺寸调大。
  - 文本设为 `游戏已保存`。
  - 启用字体阴影，颜色黑色，阴影偏移 `(0, 1)`。
- **Area2D**：
  - 碰撞层：不位于任何层。
  - 碰撞遮罩：仅检测第 5 层（玩家层）。
  - `Monitoring = true`，`Monitorable = false`。

### 2.3 提示动画

- 动画名：`游戏已保存`。
- 时长：约 3 秒。
- 关键帧：
  - 开始时 Label 可见，透明度为 0。
  - 向上弹起并淡入。
  - 在顶点完全可见。
  - 最后淡出并隐藏。
- 关键帧缓动：右键选择 `Ease In-Out` 使动画平滑。

### 2.4 脚本与图标

- 为 `SavePoint` 附加脚本，类名 `SavePoint`。
- 在编辑器图标选项中拖入保存点图标，保存后重新加载项目即可在场景树中看到自定义图标。
- 若存档点被其他节点遮挡，可调整 `Z-Index` 或层级顺序。
- > **关于保存场景 UID**：可以直接用 `owner.scene_file_path` 获取当前场景路径，但作者没有采用这种方式。原因是保存点可能被嵌套在瓦片地图层等结构下，导致 `owner` 不是预期的关卡场景；如果保存点被孤立放置（没有场景所有者），还会返回空值。教程中通过场景管理器维护当前场景 UID，更加稳健。

## 3. 消息总线系统

### 3.1 创建全局消息脚本

- 在 `Globals/` 中创建 `messages.gd` 脚本。
- 在 **项目设置 → 自动加载** 中将其添加为单例，命名为 `Messages`。
- 该脚本主要作为信号容器：

```gdscript
extends Node

signal player_interacted(player: Player)
signal player_healed(amount: float)
signal input_hint_changed(hint: String)

# 忽略未使用信号的警告
@warning_ignore("unused_signal")
signal player_interacted(player: Player)
```

- 使用 `@warning_ignore("unused_signal")` 避免控制台被未使用信号警告填满。

### 3.2 玩家发送交互信号

- 在玩家脚本中监听交互键按下事件，然后调用：

```gdscript
func _unhandled_input(event: InputEvent) -> void:
    if event.is_action_pressed("interact"):
        Messages.player_interacted.emit(self)
```

### 3.3 存档点监听交互信号

- 在 `SavePoint.gd` 中：
  - 玩家进入区域时连接 `player_interacted` 信号。
  - 玩家离开区域时断开连接。

```gdscript
func _on_player_entered(body: Node2D) -> void:
    Messages.player_interacted.connect(_on_player_interacted)

func _on_player_exited(body: Node2D) -> void:
    Messages.player_interacted.disconnect(_on_player_interacted)

func _on_player_interacted(_player: Player) -> void:
    animation_player.play("游戏已保存")
    animation_player.seek(0.0)
    SaveManager.save_game()
    Messages.player_healed.emit(99999.0)
```

## 4. 治疗玩家

### 4.1 玩家脚本中处理治疗信号

- 在玩家脚本中连接 `player_healed`：

```gdscript
func _ready() -> void:
    Messages.player_healed.connect(_on_player_healed)

func _on_player_healed(amount: float) -> void:
    hp += amount
    # 未来可在此添加治疗特效/音频
```

- 使用消息总线的好处：任何系统都可以发送治疗消息，任何系统都可以响应，无需直接引用玩家节点。

## 5. 输入提示系统

### 5.1 玩家场景结构

```
Player
├── ...
└── InputHint (Node2D)
    └── Sprite2D  # 输入图标精灵表
```

- `InputHint` 是一个 Node2D，附加独立脚本，跟随玩家移动。
- `Sprite2D` 使用输入图标精灵表，设置 `Hframes = 16`。

### 5.2 输入提示脚本

- 类名 `InputHint`，保存在 `Player/Scripts/`。
- 默认 `controller_type = "keyboard"`。
- 在 `_ready()` 中隐藏提示。

### 5.3 提示映射字典

```gdscript
const HINT_MAP := {
    "keyboard": {
        "interact": 12,  # S 键
        "attack": 10,
        "jump": 9,       # Z 键
        "dash": 11,      # C 键
        "up": 13,
    },
    "playstation": {
        "interact": 0,   # 三角
        "attack": 2,     # 方块
        "jump": 1,       # 叉
        "dash": 3,       # 圆
        "up": 4,
    },
    "xbox": {
        "interact": 3,
        "attack": 2,
        "jump": 0,
        "dash": 1,
        "up": 4,
    },
}
```

### 5.4 控制器类型检测

- 在 `_input(event)` 中检测输入设备：

```gdscript
func _input(event: InputEvent) -> void:
    if event is InputEventMouseButton or event is InputEventKey:
        controller_type = "keyboard"
    elif event is InputEventJoypadButton:
        controller_type = _get_controller_type(event.device)
        set_process_input(false)  # 检测一次后禁用，避免每帧重复检测

func _get_controller_type(device_id: int) -> String:
    var name := Input.get_joy_name(device_id).to_lower()
    if "xbox" in name:
        return "xbox"
    elif "playstation" in name or "ps" in name or "dualsense" in name or "dualshock" in name:
        return "playstation"
    # elif "nintendo" in name or "switch" in name:
    #     return "nintendo"
    return "playstation"  # 未知手柄默认显示 PlayStation 提示
```

- 注意：`set_process_input(false)` 会禁用后续检测；若需支持中途切换设备，可在进入暂停菜单等时机重新启用。

### 5.5 显示/隐藏提示

- 监听 `input_hint_changed` 信号：

```gdscript
func _on_input_hint_changed(hint: String) -> void:
    if hint == "":
        visible = false
        return
    visible = true
    var frame: int = HINT_MAP[controller_type].get(hint, 0)
    sprite.frame = frame
```

- 在存档点中发送信号：
  - 玩家进入：`Messages.input_hint_changed.emit("interact")`
  - 玩家离开：`Messages.input_hint_changed.emit("")`
