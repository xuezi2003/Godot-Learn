# C3-P3.5：错误修复与功能优化

> 本集针对社区反馈的保存系统问题进行了修复与优化，包括场景路径未初始化、新建存档未清除旧数据、保存点先保存后治疗的问题，以及输入提示系统对未知控制器的兼容处理。

## 1. 修复场景路径未初始化的问题

### 1.1 问题描述

- 直接在编辑器中按 F6 运行某个关卡场景时保存游戏，存档文件中的 `scene_path` 会是空字符串。
- 后续加载该存档时，场景管理器无法找到空资源，导致游戏卡住并报错 `找不到资源`。

### 1.2 原因分析

- `current_scene_uid` 只在 `SceneManager.transition_scene()` 中被赋值。
- 通过 F6 直接运行的场景并未经过场景管理器切换，因此 `current_scene_uid` 未被初始化。

### 1.3 修复方案

- 在 `SceneManager._ready()` 中初始化当前场景路径与 UID：

```gdscript
func _ready() -> void:
    var current_scene_path := get_tree().current_scene.scene_file_path
    current_scene_uid = ResourceUID.path_to_uid(current_scene_path)
```

- 这样即使直接运行场景，保存时也能正确记录场景 UID。

## 2. 新建存档时清除已探索区域与持久化数据

### 2.1 问题描述

- 玩家在加载一个后期存档后，如果不退出游戏直接回到标题界面开始新游戏，新的存档会继承旧存档的已探索区域和持久对象数据。

### 2.2 修复方案

- 在 `SaveManager.create_new_save_game()` 开头清空相关数据：

```gdscript
func create_new_save_game(slot: int) -> void:
    discovered_areas.clear()
    persistent_data.clear()
    current_slot = slot
    # ... 后续创建存档逻辑 ...
```

- `Array.clear()` 与 `Dictionary.clear()` 都会移除所有条目。

## 3. 保存点先治疗后保存

### 3.1 问题描述

- 当前保存点的 `on_player_interacted` 先调用 `SaveManager.save_game()`，再发送治疗信号。
- 如果玩家生命值很低，存档会记录低生命值，随后才治疗玩家，导致加载时生命值不满。

### 3.2 修复方案

- 调整顺序：先治疗玩家，再保存游戏。

```gdscript
func _on_player_interacted(_player: Player) -> void:
    Messages.player_healed.emit(99999.0)
    animation_player.play("游戏已保存")
    animation_player.seek(0.0)
    SaveManager.save_game()
```

## 4. 输入提示系统对未知控制器的兼容

### 4.1 问题描述

- 如果连接了无法识别的手柄，`controller_type` 可能被设为 `"unknown"`。
- `HINT_MAP` 中没有 `"unknown"` 键，导致访问字典时崩溃。

### 4.2 修复方案

- 将未知手柄默认映射到 PlayStation 布局：

```gdscript
func _get_controller_type(device_id: int) -> String:
    var name := Input.get_joy_name(device_id).to_lower()
    if "xbox" in name:
        return "xbox"
    # ... 其他已知类型 ...
    return "playstation"  # 未知默认
```

## 5. 添加 Nintendo Switch 布局支持

### 5.1 问题描述

- Xbox 与 Nintendo 手柄的 AB/XY 按键布局是互换的。
- 之前未处理 Switch 手柄，导致提示图标与实际按键不对应。

### 5.2 添加 Switch 映射

- 在 `HINT_MAP` 中新增 `"nintendo"` 条目：

```gdscript
"nintendo": {
    "interact": 3,  # Y
    "attack": 2,    # X
    "jump": 0,      # B
    "dash": 1,      # A
    "up": 4,
}
```

- 在 `_get_controller_type` 中检测 `"nintendo"` 或 `"switch"`：

```gdscript
elif "nintendo" in name or "switch" in name:
    return "nintendo"
```

## 6. 简化 PlayStation 检测

- 与其检查多个 PlayStation 相关字符串，不如采用反向逻辑：
  - 检测到 Xbox 或 Nintendo 则返回对应类型。
  - 否则默认返回 PlayStation。

```gdscript
func _get_controller_type(device_id: int) -> String:
    var name := Input.get_joy_name(device_id).to_lower()
    if "xbox" in name:
        return "xbox"
    elif "nintendo" in name or "switch" in name:
        return "nintendo"
    return "playstation"
```

## 7. 总结

| 问题 | 修复方式 |
|---|---|
| F6 运行场景保存后场景路径为空 | 在 `SceneManager._ready()` 中初始化当前场景 UID |
| 新存档继承旧存档的探索数据 | 在 `create_new_save_game` 中清空数组/字典 |
| 保存点先保存后治疗 | 调整执行顺序，先治疗再保存 |
| 未知控制器导致崩溃 | 默认映射到 PlayStation 布局 |
| Switch 手柄按键不对应 | 新增 Nintendo 按键映射并检测 |
