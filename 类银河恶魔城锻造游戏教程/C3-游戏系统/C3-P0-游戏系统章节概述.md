# C3-P0：游戏系统章节概述

> 本集概述了第三章将要构建的完整游戏系统，包括存档/加载、保存点、标题界面、HUD、暂停菜单与地图、门/开关、音频系统以及导出游戏，为后续功能扩展奠定技术基础。

## 本章目标

- 为游戏构建坚实的技术基础，支撑后续世界、功能与探索。
- 每个系统以“可运作部件”的形式独立实现，同时以模块化方式协同工作。
- 每个功能都代表核心机制，并留有扩展空间。

## 各集内容预告

### 第一集：全局保存/加载系统

- 创建单例模式的存档管理器（全局脚本/自动加载）。
- 实现保存、加载玩家数据，以及未来供地图系统或世界持久对象使用的数据类型。
- 奠定后续所有持久化功能的基础。

### 第二集：保存点、输入提示与消息总线

- 构建可复用的保存点场景，与存档管理器交互。
- 根据控制器类型（PlayStation / Xbox / 键盘）动态显示输入提示。
- 引入全局消息总线系统，实现组件间松耦合通信：发送者不必关心谁在监听，接收者也不必寻找发送者。

### 第三集：标题界面

- 制作标题画面，包含动态 Logo、背景与菜单。
- 提供“开始新游戏”和“从存档槽加载游戏”选项。
- 完善保存/加载循环，让用户通过标题菜单与存档槽交互。

### 第四集：玩家生命值 HUD

- 实现简单的 HUD 显示玩家生命值。
- 构建受伤时更新生命值的功能，为后续战斗做准备。

### 第五集：暂停菜单与地图系统

- 构建暂停界面。
- 在暂停菜单中集成完整地图系统：
  - 显示已探索区域。
  - 显示玩家在世界中的位置指示器。
  - 便于开发者管理和维护地图的工具。
- 在系统设置中为音量控制等功能预留位置。

### 第六集：门与开关系统

- 引入首个持续变化的游戏内机制：门与开关。
- 展示如何利用关卡门控机制释放玩家紧张感，并开启永久捷径。
- 门在激活后保持开启状态，持久化到当前会话与存档数据中。

### 第七集：音频系统

- 构建模块化音频系统，支持：
  - 音乐曲目播放与场景切换时的音乐过渡。
  - UI 音效。
  - 世界内 2D 空间音频。
- 设置多条音频总线（音乐、SFX、UI、效果通道）。
- 通过效果总线添加可配置的混响，为不同区域营造空旷感。
- 为按钮等 UI 元素提供便捷的音频连接方法。

### 第八集：导出游戏

- 导出网页版与原生操作系统版（Windows / macOS / Linux）。
- 讲解导出设置、调试日志与错误查看技巧。
- 使项目能够分享给他人进行测试与反馈。
