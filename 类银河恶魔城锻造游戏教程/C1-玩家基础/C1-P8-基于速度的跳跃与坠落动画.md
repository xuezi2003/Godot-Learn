# C1-P8：基于速度的跳跃与坠落动画

> 本集根据玩家垂直速度动态控制 `Jump` 动画的播放进度，让上升、顶点、下落的姿态自然过渡，并为下落状态补齐动画。

## 1. 核心思路

- 跳跃弧线中，速度从 `-jump_speed` 递减到 `0`，再增加到 `+max_fall_velocity`。
- 将这段速度范围重新映射到动画时间轴上。
- `Jump` 动画时长设为 1 秒：
  - `0.0 ~ 0.5` 秒：上升阶段。
  - `0.5 ~ 1.0` 秒：下落阶段。
- 动画暂停播放，由代码通过 `seek()` 直接跳转到对应时间点。

## 2. 设置 Jump 动画关键帧

按 0.1 秒间隔吸附：

| 时间 | 帧 | 姿态 |
|------|-----|------|
| 0.0s | 24 | 起跳，手臂下垂 |
| 0.1s | 25 | 上升 |
| 0.3s | 26 | 接近顶点 |
| 0.6s | 27 | 开始下落 |
| 0.8s | 28 | 高速下落 |

> 可按需添加更多帧，让过渡更细腻。

## 3. Jump 状态代码

```gdscript
func enter() -> void:
    player.animation_player.play("Jump")
    player.animation_player.pause()

func process(_delta: float) -> PlayerState:
    set_jump_frame()
    return null

func set_jump_frame() -> void:
    var frame := remap(
        player.velocity.y,
        -jump_speed,
        0.0,
        0.0,
        0.5
    )
    player.animation_player.seek(frame, true)
```

- `remap()` 将 `velocity.y` 从 `[-jump_speed, 0]` 映射到 `[0.0, 0.5]`。
- `seek(frame, true)` 强制更新动画到该时间点。

## 4. Fall 状态代码

```gdscript
func enter() -> void:
    player.animation_player.play("Jump")
    player.animation_player.pause()

func process(_delta: float) -> PlayerState:
    set_fall_frame()
    return null

func set_fall_frame() -> void:
    var frame := remap(
        player.velocity.y,
        0.0,
        player.max_fall_velocity,
        0.5,
        1.0
    )
    player.animation_player.seek(frame, true)
```

- 将下落速度从 `[0, max_fall_velocity]` 映射到 `[0.5, 1.0]`。
- 两个状态共用同一个 `Jump` 动画资源。

## 5. 最大下落速度

在 `player.gd` 中：

```gdscript
@export var max_fall_velocity: float = 600.0
```

在 `_physics_process` 中限制：

```gdscript
velocity.y += gravity * gravity_multiplier * delta
velocity.y = clampf(velocity.y, -1000.0, max_fall_velocity)
move_and_slide()
```

- 防止下落速度过快。
- 上跳最小值 `-1000` 留有余量，供跳板、冲刺等机制使用。

## 6. 效果

- 起跳时手臂/头发向后。
- 顶点时身体放松。
- 下落时头发上扬、手臂抬起。
- 从平台边缘走出也会自然进入下落姿态。
