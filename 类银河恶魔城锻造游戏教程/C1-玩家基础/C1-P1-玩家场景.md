# C1-P1：玩家场景

> 创建玩家场景、搭建测试关卡，并为角色添加基础移动、重力与输入检测。

## 1. 创建测试关卡（Playground）

- 将根节点 `Node2D` 重命名为 `Playground`。
- 添加 `Sprite2D` 作为地面视觉占位。
- 使用 `GradientTexture2D` 创建一个 32×32 像素的黑色方块。
- 为地面添加 `StaticBody2D` + `CollisionShape2D`，使用矩形碰撞形状，尺寸与精灵一致。
- 缩放并复制地面精灵，拼出基础测试关卡。

> **提示**：选中精灵后，子节点（StaticBody2D、CollisionShape2D）会一同缩放，方便快速搭建关卡。

## 2. 创建玩家场景

### 2.1 文件夹结构

在 `res://` 下新建 `Player/` 文件夹，内部再分：

```
Player/
├── Sprites/
├── Scripts/
├── Audio/
└── States/
```

### 2.2 玩家场景根节点

- 右键创建新场景，根节点类型选择 **`CharacterBody2D`**。
- 命名为 `player.tscn`。
- 脚本顶部写 `class_name Player`，继承 `CharacterBody2D`。

> **为什么用 CharacterBody2D？**
> `CharacterBody2D` 专门用于玩家控制的物理实体：它**不受物理引擎外力摆布**，但能与其他物理体交互。这样玩家不会滚下斜坡，而是保持可控移动。

### 2.3 玩家子节点

| 节点 | 作用 |
|------|------|
| `Sprite2D` | 视觉占位，使用粉色占位纹理，尺寸约 18×48 像素 |
| `CollisionShape2D` | 玩家碰撞箱，矩形，16×46 像素 |

- 将 `Sprite2D` 向上偏移 `-24` 像素，使角色底部对齐原点（脚部位置）。
- 碰撞形状也相应上移。

## 3. 第一个玩家脚本

### 3.1 `_physics_process` 与 `_process` 的区别

| 函数 | 说明 |
|------|------|
| `_process(delta)` | 按帧率调用，适合动画、UI 等 |
| `_physics_process(delta)` | 按固定物理帧率（默认 60 FPS）调用，适合移动、碰撞、重力 |

> 移动速度涉及距离/时间，必须在稳定的物理时钟下计算，因此玩家移动写在 `_physics_process` 中。

### 3.2 基础重力

```gdscript
velocity.y += gravity * delta
move_and_slide()
```

- `gravity` 设为 `980`（像素/秒²，对应项目默认 2D 重力）。
- `move_and_slide()` 会根据 `velocity` 移动角色，并在碰撞时沿表面滑动。

### 3.3 左右移动输入

```gdscript
velocity.x = 0
if Input.is_action_pressed("left"):
    velocity.x = -100
elif Input.is_action_pressed("right"):
    velocity.x = 100
```

- 使用项目设置 → 输入映射（Input Map）中预设的 `left` / `right` 动作。
- 先清零再赋值，确保无输入时停止。
