# C1-P9：总结

> 本集复盘第一章内容，讲解状态机运行原理、代码放置原则，并做一些收尾优化。

## 1. 状态机原理复盘

### 1.1 与传统状态机的区别

- 很多教程会把状态机放在独立的 `StateMachine` 节点上。
- 本系列直接将状态机写在 `player.gd` 中，因为玩家角色不需要太多模块化复用。

### 1.2 切换状态流程

```gdscript
func change_state(new_state: PlayerState) -> void:
    if new_state == null: return
    if new_state == current_state: return
    if current_state:
        current_state.exit()
    states.push_front(new_state)
    states.resize(3)
    current_state.enter()
```

1. 传入 `new_state`。
2. 如果为空或与当前状态相同，忽略。
3. 调用旧状态的 `exit()`。
4. 将新状态推到列表顶部，成为 `current_state`。
5. 调用新状态的 `enter()`。
6. `states.resize(3)` 保持历史记录最多 3 个。

### 1.3 三种调用入口

| 函数 | 调用状态函数 | 用途 |
|------|--------------|------|
| `_process(delta)` | `current_state.process(delta)` | 动画、输入结果检测、方向检测 |
| `_physics_process(delta)` | `current_state.physics_process(delta)` | 移动、速度、碰撞检测 |
| `_unhandled_input(event)` | `current_state.handle_input(event)` | 按键按下/释放 |

- 状态函数返回一个 `PlayerState`，返回 `null` 表示不切换。
- 返回值被传入 `change_state()` 执行切换。

### 1.4 为什么这样设计

- 每个状态只需关心自己的逻辑。
- 状态切换通过返回值驱动，玩家脚本统一处理。
- 避免了大量嵌套 `if`。

## 2. 代码放置原则

### 2.1 `_unhandled_input`

- 只处理按键事件。
- 引擎只在按键发生时调用，比每帧检测更高效。

### 2.2 `_physics_process`

- 影响物理体、碰撞、位移的代码放这里。
- 典型例子：`move_and_slide()`、设置速度、检测 `is_on_floor()`。

### 2.3 `_process`

- 不影响物理的代码放这里。
- 典型例子：更新动画帧、检测方向变量变化。

> 单纯设置 `velocity.x` 理论上也可放 `_process`，但作者习惯放在 `_physics_process` 中。

## 3. 重力设置讨论

- Godot 项目设置中已有 2D 重力默认值 `980`。
- 作者选择在 `player.gd` 中硬编码 `gravity`：
  - 便于查看和调试。
  - 不同实体可以有不同重力感（如漂浮敌人）。
- 若希望统一全局重力，可直接使用项目设置。

## 4. 状态机设计建议

- 避免过度设计状态转换。
- 例如：落地时无需立刻判断是否要进入 `Crouch`，先回到 `Idle`，下一帧 `Idle.process` 会自然处理下蹲输入。
- 用最少的条件实现效果。
- 条件判断比简单数值运算更耗性能。

## 5. 清理调试显示

### 5.1 移除跳跃指示器

- 注释掉 `Jump` 和 `Fall` 状态中的 `add_debug_indicator(...)` 调用。
- 保留 `add_debug_indicator` 函数，方便日后复用。

### 5.2 搜索所有调用点

- 使用 `Shift + Ctrl + F` 在项目范围内搜索。
- 确保所有调试指示器调用都已注释/删除。

## 6. 斜坡卡顿优化

### 6.1 问题

- 使用胶囊碰撞体后，某些斜坡仍可能出现 Idle / Fall 快速切换。

### 6.2 解决：地面吸附长度

在 `CharacterBody2D` 的 **Floor** 参数中：

- **Floor Max Angle**：最大斜坡角度，默认 45° 可保留。
- **Floor Snap Length**：地面吸附距离。
  - 设为 `0`：角色容易在斜坡边缘进入下落。
  - 设为 `1`：通常足够。
  - 可尝试 `2` 或 `3` 解决顽固卡顿。

> 作者最终保留 `1` 像素吸附。

### 6.3 其他 Floor 参数

- **Floor Stop On Slope**：开启，防止在斜坡上自动滑下。
- **Floor Constant Speed**：斜坡不影响移动速度。
- **Floor Block on Wall**：默认开启，防止卡墙。
