# C1-P6b：精灵表设置

> 本集为补充视频，演示如何在 Aseprite 中导入、编辑和导出角色精灵表（Sprite Sheet）。

## 1. 精灵表是什么

- 将角色所有动画帧合并到一张 PNG 中。
- 好处：只需加载一个文件，减少内存开销；Godot 可通过切换显示区域来播放动画。

## 2. 打开提供的精灵表

- 文件：`hero.png`
- 布局：4 行 × 16 列，共 128 帧。
- 单帧尺寸：80×80 像素。

## 3. 导入精灵表到 Aseprite

1. 文件 → 导入 → 导入精灵表（`Ctrl + I`）。
2. 选择导入方式：
   - **按行排列（By Rows）**：帧从左到右、再从上到下播放。
3. 设置参数：
   - 帧宽度：`80`
   - 帧高度：`80`
   - 列数：`16`
   - 行数：`4`
4. 点击导入。

## 4. 动画标签

- 帧 0–11：待机 / 行走（部分为占位帧）。
- 帧 12–19：奔跑。
- 帧 20–23：下蹲。
- 帧 24–28：跳跃。
- 帧 29：冲刺。
- 帧 30：受击。
- 帧 31–33：攻击 1。
- 帧 34–36：攻击 2。
- 帧 44–46：球体滚动。
- 后续：死亡动画。

为动画添加标签：

1. 选中连续帧。
2. 右键 → 新建标签。
3. 命名标签（如 `Run`、`Jump`）。
4. 设置循环选项。

## 5. 创建自己的角色

### 5.1 分层建议

参考作者的分层方式：

- 后臂（Back Arm）
- 腿部（Legs）
- 腰部（Waist）
- 上身（Torso）
- 头部（Head）
- 眼睛（Eyes）
- 头发（Hair）
- 前臂（Front Arm）

### 5.2 使用参考图层

- 将原始精灵表作为参考图层，调低不透明度或锁定。
- 在新建图层上绘制自己的角色。

### 5.3 一像素边距

- 作者故意让角色悬浮在地面像素上方，四周留出一像素边距。
- 目的：避免 Canvas Texture 渲染时出现色渗（颜色渗出到相邻像素）。

## 6. 导出精灵表

1. 文件 → 导出 → 导出精灵表。
2. 设置：
   - 排列方式：按行（By Rows）。
   - 固定列数：`16`（便于跟随教程）。
   - 图层：全部可见图层 / 全部图层。
   - 输出到文件：勾选。
3. 保存为 `hero.png` 或其他文件名。

> 如果不想覆盖原始文件，请修改导出文件名。

## 7. 课后作业

- 在 Aseprite 中处理 `hero.png` 或创建自己的角色。
- 熟悉精灵表布局，为下一集导入 Godot 做准备。
