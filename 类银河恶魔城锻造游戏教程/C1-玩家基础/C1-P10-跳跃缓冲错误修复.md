# C1-P10：跳跃缓冲错误修复

> 修复跳跃缓冲与可变高度跳跃冲突导致的 bug：缓冲跳跃时玩家只能跳到最大高度，无法实现短跳。

## 1. Bug 描述

- 玩家在落地前按下跳跃键触发 Jump Buffer。
- 落地后起跳，但即使快速松开跳跃键，角色仍会跳到最高高度。
- 原因是：跳跃键在下落状态中已经被释放，进入 `Jump` 状态后不会再触发 `is_action_released`，可变高度跳跃逻辑被跳过。

## 2. 复现方法

- 将 `Engine.time_scale` 设为 `0.5` 放慢游戏。
- 将 `jump_buffer_time` 调到 `1.0` 秒左右。
- 起跳后快速松开，在落地前再次轻点跳跃键。

## 3. 方案一：落地时必须按住跳跃键

在 `fall_state.gd` 中：

```gdscript
func physics_process(_delta: float) -> PlayerState:
    if player.is_on_floor():
        if buffer_timer > 0 and Input.is_action_pressed("jump"):
            return jump
        return idle
    return null
```

- 只有落地时仍按住跳跃键才会触发缓冲跳跃。
- 简单，但要求玩家保持按住，手感稍严格。

## 4. 方案二：追溯处理按键释放（推荐）

### 4.1 恢复原始缓冲检查

```gdscript
if buffer_timer > 0:
    return jump
```

### 4.2 在 Jump 状态进入时处理

```gdscript
# jump_state.gd
func enter() -> void:
    player.velocity.y = -jump_speed
    player.animation_player.play("Jump")
    player.animation_player.pause()

    if player.previous_state == fall and not Input.is_action_pressed("jump"):
        await get_tree().physics_frame
        player.velocity.y *= 0.5
        player.change_state(fall)
```

### 4.3 关键点

- 判断之前状态是 `Fall`，且当前未按住跳跃键 → 这是一次缓冲跳跃且按键已释放。
- 等待一个物理帧，避免状态循环。
- 将 `velocity.y` 减半，模拟短跳效果，并切换到 `Fall`。

## 5. 修复另一个相关问题

### 5.1 问题

- 穿过单向平台后，可能残留 `buffer_timer`，导致不期望的二次跳跃。

### 5.2 解决

在 `fall_state.gd` 的 `exit()` 中重置缓冲计时器：

```gdscript
func exit() -> void:
    player.gravity_multiplier = 1.0
    buffer_timer = 0.0
```

- 确保每次离开 `Fall` 时清除缓冲状态。

## 6. 总结

- 缓冲跳跃的实现需要考虑按键释放时机。
- 使用 `previous_state` 和 `Input.is_action_pressed` 可以判断是否需要追溯处理。
- `exit()` 函数是清理状态、重置计时器的好地方。
