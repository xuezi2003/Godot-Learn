# C1-P2：玩家状态机

> 本集在玩家脚本内构建一个有限状态机（Finite State Machine），把玩家行为拆分为 Idle、Run 等独立状态，使后续功能更易于扩展。

## 1. 为什么需要状态机

### 1.1 不用状态机的问题

如果所有逻辑都塞在 `_physics_process` 中：

- 需要大量嵌套 `if` 判断：是否在跳跃？是否冲刺？能否冲刺？是否在攻击？
- 条件越叠越复杂，容易出错。
- 新增能力时代码耦合严重。

### 1.2 状态机的优势

- 每个状态独立管理自己的输入、物理与逻辑。
- 一次只运行一个状态，代码清晰。
- 新增能力只需新增一个状态节点/脚本。

### 1.3 状态机的代价

- 初期代码量增加。
- 部分代码会在多个状态中重复（如输入检测）。
- 结构固定，某些复杂组合需要额外设计。

## 2. 状态机设计

- 状态机直接集成在 `player.gd` 中，不单独做成一个节点模块。
- 每个状态是一个 `Node` 子节点，挂载自己的状态脚本。

```
Player (CharacterBody2D)
└── States (Node)
    ├── Idle
    ├── Run
    └── ...
```

## 3. 创建 `PlayerState` 基类

### 3.1 文件位置

`Player/States/00_player_state.gd`

### 3.2 类结构

```gdscript
@icon("res://Player/States/state_icon.svg")
class_name PlayerState
extends Node

#region 引用
var player: Player
var next_state: PlayerState
#endregion

## 生命周期函数
func init() -> void: pass
func enter() -> void: pass
func exit() -> void: pass

## 每帧调用
func process(delta: float) -> PlayerState: return null
func physics_process(delta: float) -> PlayerState: return null

## 输入
func handle_input(event: InputEvent) -> PlayerState: return null
```

- `@icon`：为继承此类的节点在场景树中显示自定义图标，便于快速识别状态节点。
- `#region` / `#endregion`：GDScript 的代码折叠标记，可将相关变量折叠起来，保持脚本整洁。

### 3.3 说明

- `init()`：状态初始化。
- `enter()`：进入状态时调用（播放动画、重置变量等）。
- `exit()`：退出状态时调用（清理计时器、重置变量等）。
- `process` / `physics_process` / `handle_input` 返回 `PlayerState`，用于触发状态切换；返回 `null` 表示不切换。

### 3.4 状态引用

在 `PlayerState` 中声明对其他状态节点的引用，方便各状态直接返回目标状态：

```gdscript
@onready var idle: PlayerState = %Idle
@onready var run: PlayerState = %Run
```

> 使用 `%` 唯一名称访问，即使移动节点位置也不会断开引用。

## 4. 玩家脚本中的状态机

### 4.1 状态记录

```gdscript
var states: Array[PlayerState]
var current_state: PlayerState:
    get: return states.front()
var previous_state: PlayerState:
    get: return states[1] if states.size() > 1 else null
```

- `states` 最多保留 3 个状态，用于追踪当前、上一个、上上一个状态。
- > GDScript 数组索引从 `0` 开始，`states[0]` 是第一个元素，`states[1]` 是第二个元素，即前一个状态。

### 4.2 初始化状态

```gdscript
func _ready() -> void:
    initialize_states()

func initialize_states() -> void:
    states = []
    for child in $States.get_children():
        if child is PlayerState:
            child.player = self
            states.append(child)
    if states.is_empty(): return
    for state in states:
        state.init()
    change_state(current_state)
```

### 4.3 切换状态

```gdscript
func change_state(new_state: PlayerState) -> void:
    if new_state == null: return
    if new_state == current_state: return
    if current_state:
        current_state.exit()
    states.push_front(new_state)
    states.resize(3)
    current_state.enter()
```

### 4.4 每帧调用当前状态

```gdscript
func _process(delta: float) -> void:
    change_state(current_state.process(delta))

func _physics_process(delta: float) -> void:
    change_state(current_state.physics_process(delta))

func _unhandled_input(event: InputEvent) -> void:
    change_state(current_state.handle_input(event))
```

- > `move_and_slide()` 会在内部自动处理 `delta`，因此不要在调用它之前将速度乘以 `delta`。只有重力等需要显式按帧累积的项才需要乘 `delta`。
