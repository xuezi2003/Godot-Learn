# C0-P1：概览与规划

> 介绍项目定位、核心特性清单与系列章节规划。

## 1. 项目概述

- **系列名称**：类银河战士恶魔城锻造工坊
- **开发引擎**：Godot 4（教程使用 4.5 版本）
- **脚本语言**：GDScript
- **游戏类型**：2D 像素风动作平台 + 类银河战士恶魔城（Metroidvania）
- **目标**：从零开始打造一款可玩的、探索驱动的小型 Metroidvania 游戏，重点在于扎实的手感、清晰的代码结构以及完整的发布流程。

与作者之前的 ARPG 系列相比，本系列会更加聚焦：

1. **更透明的规划过程**：提前公开决策思路、开发流程与章节安排。
2. **模块化推进**：分块构建玩家、世界、系统，每个模块稳定后再进入下一环节。
3. **减少迂回**：尽量避免边做边改、依赖临时反馈的迭代方式。

### 作者背景与选题缘由

主讲者 Michael 对类银河战士恶魔城类型有很深的感情，这也是本系列的选题原因之一。他曾在 Unity 中参加过多次 Metroidvania 主题的游戏创作活动，陆续完成过一些小规模作品，例如《台原》《23 号休息站》《英雄的废品》《千禧战士》等。其中《千禧战士》是他独立完成的项目，常被拿来作为功能实现参考。这些经历让他对如何构建此类游戏有一定经验，也希望在本系列中超越过去的做法。


## 2. 什么是类银河战士恶魔城游戏

### 2.1 定义与边界

把项目定义为 Metroidvania，实际上是给自己划定一个可自由创作的框架（约束即沙盒）。它有助于：

- 明确目标与探索方向（美术风格、核心机制、功能范围）。
- 在开发中遇到不确定决策时，回到这个定义判断“这是否真的需要”。
- 管理优先级，避免范围无限膨胀。

### 2.2 核心特征

典型的 Metroidvania 通常包含：

- **2D 动作平台**：以 2D 横版闯关为主要表现形式。
- **非线性探索**：玩家可以在世界中自由探索，但部分区域需要获得新能力后才能进入。
- **成长体系与能力锁**：角色的成长和世界探索都通过解锁新能力来驱动。


## 3. 核心目标与特性清单

> 以下清单是作者为该项目构思的初版功能集合，**并非一成不变**。建议观众结合自身项目做增删。

### 3.1 玩家能力

| 能力 | 说明 |
|------|------|
| 奔跑 / 跳跃 / 下蹲 | 平台游戏基础操作；下蹲可作为可选能力。 |
| 攻击 | 构建基础连击系统，作为起点。 |
| 二段跳 | 到达更高区域，典型 Metroidvania 能力锁设计。 |
| 冲刺 | 快速位移能力。 |
| 变形翻滚（Morph Ball） | 受《银河战士》启发，虽然整体风格更偏向恶魔城，但仍作为有趣能力纳入候选。 |
| 地面重击 / 敌弹反 | 可作为扩展攻击手段或平台解谜工具。 |
| 贴墙滑行 / 蹬墙跳 | 高级移动技巧，系列中未必全部实现。 |

### 3.2 敌人设计

- **基础行走敌人**：如僵尸、骷髅等，沿路径移动。
- **固定炮台 / 机关**：向玩家发射或投掷抛射物。
- **飞行敌人**：如美杜莎之首类飞行单位。
- **基础 AI 敌人**：比行走敌人稍聪明，会追击玩家，而非单纯直线巡逻。

### 3.3 Boss 设计

Metroidvania 中常见的两种 Boss 类型：

1. **大型关卡式 / 场景型 Boss**
   - 本身就是关卡的一部分。
   - 体型巨大，玩家主要躲避定时触发的攻击。
   - 攻击前通常有明显预兆。
2. **类玩家小型 Boss**
   - 战斗体验接近 PvP。
   - Boss 拥有与玩家对等的移动、跳跃、攻击、冲刺能力。

### 3.4 关卡与探索机制

- **能力门槛**：用新能力封锁/解锁区域。
- **秘密与隐藏区域**：提升探索成就感。
- **安全区域**：可存档并恢复生命值。
- **可解锁通道**：与秘密区域、捷径结合。
- **宝箱、移动平台、单向平台、环境危害**：基础关卡元素。

### 3.5 游戏系统

- 玩家生命值（可扩展为更多属性）。
- 存档 / 读档系统。
- 玩家能力获取机制。
- 生命值升级 / 持久化记录（例如门一旦打开就保持开启、击败的 Boss 不再刷新）。
- 敌人生成系统。
- 对话与过场动画系统（可能复用 ARPG 系列中的方案）。

### 3.6 UI 与界面

- 暂停菜单（Pause Menu）
- 地图系统（带 Metroidvania 风格的房间地图）
- 标题画面（Title Screen）
- 游戏结束画面（Game Over Screen）

### 3.7 润色与手感

- 视觉特效与粒子特效
- 音效、背景音乐、小型原声专辑（计划 3~4 首曲目）
- 操作手感打磨：触感、趣味性、奖励感


## 4. 项目约束与规模

| 维度 | 约束 |
|------|------|
| 引擎 | Godot 4 + GDScript |
| 画面 | 2D 像素艺术（作者提供基础素材，也可使用高清风格，但设置需自行调整） |
| 美术工具 | Aseprite（非强制，可用其他像素画工具） |
| 音乐 | 创作一张小型原声专辑（3~4 首） |
| 平台 | 网页版、PC、Mac、Linux |
| 规模 | 小游戏，非商业体量；约 **10~15 个房间**，约 **2 场 Boss 战** |

> 这些约束是为教程系列服务的。如果是个人项目，可以根据自己的目标调整，但初次尝试建议控制规模。


## 5. 系列章节规划

| 章节 | 主题 | 主要内容 |
|------|------|----------|
| **序章** | 项目引言与环境配置 | 规划、Godot 项目搭建、GitHub 仓库配置。 |
| **第一章** | 核心玩家机制 | 搭建玩家状态机，实现移动、跳跃、攻击等基础能力。 |
| **第二章** | 世界构建基础 | TileMap、房间设计、房间过渡、能力锁、环境危害。 |
| **第三章** | 世界游戏系统 | 存档/读档、音乐播放、暂停菜单、地图系统。 |
| **第四章** | 高级玩家机制 | 二段跳、冲刺、变形翻滚等能力的完善。 |
| **第五章** | 敌人与 AI | 基础敌人类型、伤害系统、敌人掉落、基础 AI 敌人。 |
| **后续** | Boss 战与发布准备 | 首领战、打磨、多平台导出与发布。 |

> 这是一份动态文档，可能会根据实际教学需要或社区反馈调整。


## 6. 课后作业

1. 参考上述清单，**为你的游戏列出一份属于自己的目标与特性清单**。
2. 思考你的 Metroidvania 游戏想要呈现的核心体验、美术风格与机制重点。
3. 把这份计划写下来（文档、便签、餐巾纸均可），在后续开发中作为决策依据。

> “即使我没有在视频中教你实现某个具体功能，本系列也会覆盖你需要掌握的基础知识和技能，让你有能力自行扩展。”
