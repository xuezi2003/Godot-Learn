# C0-P2：项目搭建

> 在 Godot 4.5 中创建项目，配置显示、纹理与输入映射。

## 1. 下载与安装 Godot

- **教程版本**：Godot 4.5（建议使用视频发布时的最新稳定版）
- **下载地址**：[Godot 官网](https://godotengine.org/) → 点击 “Download Latest” → 选择适合自己系统的版本
- **Windows 用户**：通常选择 **x86_64** 版本
- 下载完成后解压即可使用，可在桌面创建快捷方式


## 2. 创建新项目

1. 启动 Godot 后，在项目管理器中点击 **“创建新项目”**。
2. 设置项目名称，例如：`银河恶魔城锻造`。
3. 选择保存位置（默认在文档文件夹，建议自定义到专门的开发目录）。
4. **渲染器**：条件允许时选择 **Forward+**；若硬件不支持，可改用 **Compatibility**。
5. **版本控制元数据**：勾选 **Git**，这会生成 `.gitignore` 等仓库所需文件。
6. 点击 **创建并编辑**，Godot 会自动打开编辑器。


## 3. 创建首个 2D 场景并设为主场景

### 3.1 创建场景

- 默认打开的是 3D 编辑器视图。
- 点击 **2D 场景** 创建一个新的 2D 根节点。
- 保存场景：`Ctrl + S`，命名为 `游乐场.tscn`（或 `playground.tscn`），暂存于项目根目录。

### 3.2 设置主场景

1. 将 Godot 图标节点拖入场景（仅作占位/测试用）。
2. 按 `F5` 运行项目。
3. 首次运行会提示选择 **主场景（Main Scene）**，选择当前 `游乐场.tscn` 并确认。
4. 之后每次 `F5` 都会默认运行该场景。


## 4. 项目设置

打开路径：`项目 → 项目设置`（`Project → Project Settings`）。

### 4.1 开启高级设置

- 项目设置窗口右上角勾选 **“高级设置”**（Advanced Settings）。
- 部分参数在基础模式下不可见，本系列中大多需要高级模式。

### 4.2 显示 / 窗口 / 拉伸

#### 4.2.1 分辨率与窗口尺寸

| 设置项 | 推荐值 | 说明 |
|--------|--------|------|
| 视口宽度（Viewport Width） | `480` | 游戏逻辑分辨率宽 |
| 视口高度（Viewport Height） | `270` | 游戏逻辑分辨率高 |
| 窗口宽度覆盖（Window Width Override） | `1440` | 实际窗口显示宽度 |
| 窗口高度覆盖（Window Height Override） | `810` | 实际窗口显示高度 |

- `480 × 270` 是游戏逻辑分辨率；窗口覆盖 `1440 × 810` 是其 **3×** 整数倍放大，便于像素艺术清晰显示。
- 窗口覆盖尺寸仅改变窗口大小，不改变渲染分辨率；视口会自动缩放适应窗口。
- 如果你偏好高清非像素风格，可以直接将视口设为 `1920 × 1080` 或 `2K`，并调整相关设置。

#### 4.2.2 拉伸模式

- 路径：`显示 → 窗口 → 拉伸`
- **拉伸模式（Stretch Mode）**：本教程选择 **Canvas Items**
  - 优点：像素艺术在屏幕上呈现更流畅、清晰的 2D 图形。
  - 与 **Viewport** 模式的区别：Viewport 会强制以低分辨率渲染再放大，画面更“硬像素”；Canvas Items 是另一种适合像素艺术的方案。
- 视频中简短演示了两种模式切换后的画面差异：Viewport 模式下图标边缘更像素化，Canvas Items 模式下稍显平滑。最终教程采用 Canvas Items。
- 若使用高清艺术或想要不同效果，可切换为 Viewport 或其他模式。

### 4.3 渲染 / 纹理 / 默认纹理过滤器

- 路径：`渲染 → 纹理`
- **默认纹理过滤器（Default Texture Filter）**：像素艺术选择 **Nearest**（最近邻）
  - **Linear（线性）**：放大时平滑纹理边缘，适合高清风格。
  - **Nearest（最近邻）**：保留硬边像素，适合像素艺术。
- 将项目默认设为 Nearest 后，新导入的素材会自动继承该设置，无需逐个手动调整。
- 单个 Sprite 节点的 `Texture → Filter` 可覆盖为 Linear 等，但项目默认应保持 Nearest。
- > **注意**：若项目默认保留 Linear，所有像素素材导入后会显得模糊/发虚，必须逐个手动改为 Nearest 才能修正。因此像素艺术项目应在一开始就把默认纹理过滤器设为 Nearest。


## 5. 输入映射（Input Map）

打开路径：`项目 → 项目设置 → 输入映射`（`Input Map`）。

### 5.1 为什么要提前配置

- 在开发过程中随时引用统一命名的动作，代码可读性更高。
- 避免实现某个功能时才临时去添加输入绑定。

### 5.2 添加动作与绑定

#### 5.2.1 方向移动

| 动作名 | 键盘绑定 | 手柄绑定（示例） |
|--------|----------|------------------|
| `left` | 方向键 ← / A | 左摇杆向左 / 方向键左 |
| `right` | 方向键 → / D | 左摇杆向右 / 方向键右 |
| `up` | 方向键 ↑ / W | 左摇杆向上 / 方向键上 |
| `down` | 方向键 ↓ / S | 左摇杆向下 / 方向键下 |

#### 5.2.2 能力动作

| 动作名 | 说明 | 键盘绑定（模仿《空洞骑士》） | 手柄绑定 |
|--------|------|------------------------------|----------|
| `jump` | 跳跃 | `Z` / `Space` | 主讲者按个人习惯绑定 |
| `attack` | 攻击 | `X` | 主讲者按个人习惯绑定 |
| `dash` | 冲刺 | `C` | 主讲者按个人习惯绑定 |
| `action` | 互动/动作 | `S` | 主讲者按个人习惯绑定 |

> 手柄按键部分原字幕语音识别较模糊，仅表达“使用 PlayStation 手柄并按个人习惯为跳跃、攻击、冲刺、动作分别绑定按键”。建议按自己手柄布局配置。

### 5.3 绑定方法

1. 在输入映射中点击 **“添加新动作”**，输入动作名（如 `jump`）。
2. 点击动作右侧的 **“+”** 添加事件。
3. 按下键盘按键或手柄按键即可自动识别并绑定。
4. 同一个动作可绑定多个输入事件（例如 `jump` 同时绑定 `Z` 和 `Space`）。


## 6. 课后作业

1. **设计你的项目图标**：
   - 创建或找一张能代表项目的图片（PNG / JPG / SVG 均可）。
   - 放入项目文件夹。
   - 在 `项目设置 → 常规 → 应用程序 → 配置 → 图标` 中指定该图片。
2. 确认项目能正常运行，窗口尺寸与纹理过滤设置符合你的美术风格。
3. 根据个人习惯调整输入映射（不必完全照搬示例）。
