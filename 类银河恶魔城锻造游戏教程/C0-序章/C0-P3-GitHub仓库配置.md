# C0-P3：GitHub 仓库配置

> 用 GitHub Desktop 为 Godot 项目建立仓库，演示提交、推送、拉取。

## 1. 准备工作

### 1.1 注册 GitHub 账号

- 访问 [github.com](https://github.com/)。
- 注册一个免费账号（免费用户即可创建私有仓库）。
- 仓库可设为 **私有**（仅自己可见）或 **公开**（任何人可见）。

### 1.2 下载 GitHub Desktop

- 访问 GitHub Desktop 官网下载并安装。
- 教程使用图形化客户端，便于初学者理解版本控制流程；熟悉命令行的用户也可使用 `git` 命令行工具。
- 安装完成后登录你的 GitHub 账号。


## 2. 在本地创建仓库

### 2.1 选择项目文件夹

1. 打开 GitHub Desktop。
2. 点击 **文件 → 新建仓库**（`File → New Repository`）。
3. 点击 **本地路径** 浏览并选择你的 Godot 项目根目录。
   - 该目录下应包含 `project.godot` 文件。
   - 由于在创建项目时勾选了 Git 元数据，目录中已存在 `.gitignore` 等初始 Git 文件。
   - 主讲者本地有两个版本：一个是提前开发的“开发版”，一个是录制用的“教程版”；演示时选择的是教程版项目文件夹。
4. **名称**：保持与项目文件夹同名（如 `metroidvania-forge`）。
   - 注意：不要额外创建子文件夹，否则仓库会被嵌套一层。

### 2.2 初始化选项

| 选项 | 建议 |
|------|------|
| 使用 README 初始化 | ✅ 勾选，方便在仓库首页写项目说明 |
| Git Ignore | 选择 **Godot**，自动忽略 `.godot/`、导入缓存等本地-only 文件 |
| License | 根据个人需要选择，本教程未使用 |

点击 **创建仓库** 后，GitHub Desktop 会完成本地 Git 仓库初始化。


## 3. 发布到 GitHub

1. 创建本地仓库后，GitHub Desktop 会显示 **“发布仓库”（Publish Repository）** 按钮。
2. 点击后设置：
   - **名称**：默认使用本地仓库名。
   - **描述**：可选，写一句项目简介。
   - **保持代码私有**：勾选则为私有仓库，取消则为公开仓库。
   - **组织**：选择要关联的 GitHub 账号/组织，一般选 “None”。
3. 点击 **发布仓库**，项目文件会被上传到 GitHub。
4. 上传完成后，可在 GitHub 网页端查看仓库。


## 4. 日常同步流程

### 4.1 修改项目并查看差异

1. 在 Godot 中为项目添加图标：
   - 右键项目 → **在文件管理器中打开**。
   - 将制作好的图标图片拖入项目文件夹。
   - 回到 Godot，`项目设置 → 常规 → 应用程序 → 配置 → 图标`，选择新图标。
   - 删除默认的 Godot 图标节点（可选）。
2. 保存项目后，切换到 GitHub Desktop。
3. 在 GitHub Desktop 中可看到文件变更：
   - **红色减号**：被删除的文件。
   - **绿色加号**：新增的文件（如图标 PNG）。
   - **黄色标记**：被修改的文件（如 `.import` 缓存文件）。

### 4.2 提交（Commit）

1. 在 GitHub Desktop 左下角填写 **提交摘要**（如：`更新项目图标`）。
2. 可选：填写更详细的描述。
3. 点击 **提交到主分支**（Commit to main）。

> 提交只是把变更记录到本地 Git 历史，此时还没有上传到 GitHub 云端。

### 4.3 推送（Push）

1. 提交完成后，点击 **推送至原端**（Push to origin）。
2. 这会把本地的提交上传到 GitHub 服务器。
3. 推送完成后，刷新 GitHub 网页即可看到最新变更。

### 4.4 拉取远程变更（Pull）

如果先在 GitHub 网页端修改了文件（例如编辑 README）：

1. 本地仓库会显示与远程不同步。
2. 在 GitHub Desktop 中点击 **获取原端**（Fetch origin）。
3. 检测到远程有新提交后，点击 **从原端拉取**（Pull from origin）。
4. 本地文件会更新为远程最新版本。
5. 然后再推送本地的新提交。

> 独自一人工作时，通常只需 **提交 → 推送**。与他人协作时，才需要频繁地 **获取 → 拉取 → 处理冲突 → 推送**。


## 5. 查看提交历史

- 在 GitHub 网页端点击仓库首页的提交数量链接，可查看完整提交历史。
- 点击某次提交，可查看该次提交改动了哪些文件、具体修改内容。
- 通过历史记录，可以随时回溯到项目任意版本，便于排查“我改坏了什么”。


## 6. 课后作业

1. 为自己在 C0-P2 中创建的 Godot 项目配置 GitHub 仓库。
2. 设计并添加项目图标，完成一次提交与推送，确认 GitHub 网页端能看到更新。
3. 在 GitHub 上编辑一次 README，然后在 GitHub Desktop 中练习 **获取 → 拉取** 流程。
4. 如果你是 Godot 新手，花些时间探索编辑器界面，随意添加节点、运行项目，熟悉基本操作。
