# C5-P1：基础史莱姆敌人

> 本集用硬编码脚本实现第一个敌人——蓝色史莱姆，涵盖动画、碰撞、伤害/危险区域、边缘检测、击退等最基础的敌人要素。


## 1. 史莱姆的功能清单

- 左右移动，受重力影响。
- 撞墙或检测到边缘时转身。
- 可设置初始面向方向。
- 对玩家造成伤害（危险区域）。
- 可被玩家攻击并扣血（伤害区域）。
- 受击时有击退效果。
- 死亡后播放死亡动画并移除。


## 2. 场景结构

```
Slime (CharacterBody2D)
├── Sprite2D
├── CollisionShape2D          # 身体碰撞（胶囊，旋转后放平）
├── AnimationPlayer
├── HazardArea (Area2D)         # 对玩家造成伤害
│   └── CollisionShape2D
└── DamageArea (Area2D)         # 被玩家攻击
    └── CollisionShape2D
```


## 3. 碰撞层配置

| 节点 | Layer | Mask | 说明 |
|------|-------|------|------|
| `Slime` 根节点 | 无 | 地面 + 单向平台 | 敌人本身不与玩家发生物理碰撞 |
| `HazardArea` | 敌对层 | 玩家层 | 检测玩家并造成伤害 |
| `DamageArea` | 受伤层 | 玩家攻击层 | 接收玩家攻击 |

> 注意：敌人根节点**不要**放在地面层，否则玩家会被挡住。


## 4. 动画设置

| 动画名 | 帧范围 | 循环 | 时长 |
|--------|--------|------|------|
| `walk` | 0–5 | 是 | 约 0.6 s（10 FPS） |
| `stun` | 12–16 | 否 | 0.5 s |
| `death` | 6–11（可能含空白帧 17） | 否 | 0.6–0.7 s |

- 受击音效放在 `DamageArea` 上。
- 死亡音效放在史莱姆脚本里。


## 5. 核心脚本逻辑（Slime）

### 导出变量

```gdscript
@export var health: float = 3.0
@export var move_speed: float = 30.0
@export var face_left_at_start: bool = false
@export var death_audio: AudioStream
```

### 关键变量

- `direction: float`：移动方向，`1` 向右，`-1` 向左。
- `move_tween: Tween`：用于击退效果。

### 物理移动

```gdscript
func _physics_process(delta: float) -> void:
    velocity.x = direction * move_speed
    velocity += get_gravity() * delta
    move_and_slide()
```

### 改变方向

- 撞墙时调用 `change_direction(-direction)`。
- 同时翻转 `Sprite2D.flip_h`。

### 边缘检测器（EdgeDetector）

- 新脚本，继承 `RayCast2D`。
- 自动设置 Mask 为地面层 + 单向平台层。
- 发出 `edge_detected` 信号。
- 跟随敌人方向自动翻转位置。

### 受击与死亡

- 连接 `DamageArea.damage_taken` 信号。
- 扣减生命值；大于 0 播放 `stun`，否则播放 `death`。
- 死亡瞬间立刻释放 `DamageArea` 与 `HazardArea`，避免继续交互。
- 动画结束信号：
  - `stun` → 回到 `walk`
  - `death` → `queue_free()`

### 击退（Knockback）

- 根据攻击者位置判断方向。
- 用 `Tween` 临时把 `direction` 拉到反向，再慢慢恢复。
- 若敌人已死亡，方向保持为 0，停止移动。


## 6. 注意事项

- 胶囊碰撞体稍微缩小，避免卡在单像素角落。
- `move_speed` 与动画速度要匹配，否则看起来像滑冰。
- 这是一个“硬编码”敌人示例，适合少量敌人；大量敌人时需要更模块化的方案。
