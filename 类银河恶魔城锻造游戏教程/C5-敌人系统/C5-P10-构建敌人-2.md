# C5-P10-2：构建敌人（中）

> 本集继续展示 Demon Chest 的飞行决策引擎与飞行攻击状态代码细节，强调状态的可复用性。


## 1. FlyingDecisionEngine

- 与 `DecisionEngine_BasicAttack` 几乎相同。
- 所有状态通过导出变量手动分配，便于查看哪些状态已配置。

```gdscript
@export var fly_state: EnemyState
@export var chase_state: EnemyState
@export var attack_state: ES_Attack
@export var stun_state: EnemyState
@export var death_state: EnemyState
```

决策逻辑：

1. 死亡 → 死亡状态。
2. 受伤 → 眩晕状态。
3. 无法决策 → `null`。
4. 有目标：
   - 能攻击 → 攻击状态。
   - 否则 → 追逐状态。
5. 无目标 → 飞行/待机状态。


## 2. ES_FlyAttack（飞行攻击）

### 与 ES_Attack 的异同

- 相同：
  - 使用 `attack_range`、`cooldown`、`can_attack()`。
  - 通过动画时长决定状态持续时间。
- 不同：
  - 速度是 `Vector2`。
  - 方向每帧重新计算并指向玩家。
  - 使用 `speed_curve` 控制冲刺强度。

### 代码示例

```gdscript
func physics_update(delta: float) -> void:
    timer += delta
    var speed_sample := 1.0
    if speed_curve:
        speed_sample = speed_curve.sample(timer / duration)

    var target = blackboard.target
    if target:
        var direction = enemy.global_position.direction_to(target.global_position)
        enemy.change_direction(sign(direction.x))
        enemy.velocity = direction * move_speed * speed_sample
```


## 3. ES_Fly（飞行/待机）

- 播放飞行动画。
- 仅设置 X 方向速度。
- 可作为巡逻状态使用。


## 4. 死亡处理

- Demon Chest 没有专门死亡动画，使用淡出效果。
- 在 `ES_Death` 中可用 `Tween` 调低 `modulate.a` 至 0 后 `queue_free()`。


## 5. 关键经验

- 状态应尽量通用：飞行与行走只是速度维度不同。
- 决策引擎保持简单：if-else 链即可满足大多数需求。
- 通过导出变量分配状态，比 `@onready var` 硬编码更安全、可读。
