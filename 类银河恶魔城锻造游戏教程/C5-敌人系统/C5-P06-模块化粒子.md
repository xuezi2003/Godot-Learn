# C5-P06：模块化粒子

> 本集实现一个即插即用的 `EnemyHitParticles` 节点，命中敌人时自动发射粒子，死亡时发射另一组粒子，无需额外脚本连接。


## 1. 设计目标

- 拖到敌人身上即可生效。
- 自动连接敌人的 `hit` 与 `killed` 信号。
- 支持普通命中粒子和死亡粒子两组效果。
- 即使所有者不是 `Enemy`，只要作为 `DamageArea` 的兄弟节点也能工作。


## 2. 场景与脚本

```
EnemyHitParticles (Marker2D)
```

### 导出变量

```gdscript
@export var hit_particles: Array[ParticleSettings]
@export var death_particles: Array[ParticleSettings]

var enemy_was_killed: bool = false
```


## 3. 两种连接方式

### 情况 A：所有者是 Enemy

```gdscript
if owner is Enemy:
    owner.hit.connect(_on_hit)
    owner.killed.connect(_on_killed)
```

### 情况 B：作为 DamageArea 的兄弟节点

```gdscript
else:
    for c in owner.get_children():
        if c is DamageArea:
            c.damage_taken.connect(_on_hit)
```

> 死亡信号需要其他机制（如可破坏物的 `destroyed` 信号）扩展。


## 4. 命中/死亡处理

### `_on_killed()`

- `enemy_was_killed = true`

### `_on_hit(attack_area: AttackArea)`

1. 计算方向：
   ```gdscript
   var direction = (global_position - attack_area.global_position).normalized()
   direction.x *= -1   # 修正朝向
   ```
2. 选择粒子数组：
   - 若 `enemy_was_killed` 为 `true` → `death_particles`
   - 否则 → `hit_particles`
3. 遍历粒子设置，调用全局粒子系统发射。


## 5. 配置示例（史莱姆）

| 类型 | 粒子数 | 颜色 |
|------|--------|------|
| 命中粒子 1 | 24 | 蓝色 |
| 命中粒子 2 | 8 | 白色 |
| 死亡粒子 | 8 | 紫色/亮色 |

- 使用带透明度渐变的 8×8 圆形纹理，使粒子更明显。
- 将 `EnemyHitParticles` 放在敌人中心略上方。


## 6. 关键细节

- `Enemy` 会先发出 `killed` 信号，再发出 `hit` 信号；这样死亡时粒子系统能切换为死亡粒子。
- 方向计算包含 Y 轴，攻击敌人上方时粒子会向上飞溅。
- 若方向反了，把 `direction.x *= -1` 修正。
