# C5-P08：智能巡逻攻击敌人

> 本集以社区投稿的 **Living Armor（活体盔甲）** 为例，实现一个会巡逻、发现玩家后追逐、进入攻击范围后攻击的敌人，并介绍攻击状态与攻击区域（AttackArea）的配合。


## 1. 敌人目标

- 普通状态：沿平台巡逻（行走）。
- 发现玩家：进入追逐状态，朝玩家移动。
- 接近玩家：进入攻击状态，播放攻击动画并造成伤害。
- 内置攻击冷却，避免连续攻击。


## 2. Living Armor 场景准备

- 新建文件夹 `enemies/living_armor/`。
- 复制 `Slime2` 场景作为起点，替换精灵为 `living_armor`。
- 若素材默认朝左，将 `Sprite2D.scale.x = -1` 翻转为朝右（本系列约定敌人默认朝右）。
- 调整碰撞形状、伤害区域、危险区域、视野锥大小。


## 3. 动画设置

| 动画名 | 帧范围 | 说明 |
|--------|--------|------|
| `idle` | 0–5 | 待机，循环 |
| `walk` | 14–32 | 行走，循环 |
| `attack` | 33–47 | 蓄力后斩击 |
| `stun` | 48–49 | 受击停顿（复用死亡前两帧） |
| `death` | 48 往后 | 倒地死亡 |


## 4. 新状态

### ES_Chase（追逐）

```gdscript
class_name ES_Chase
extends EnemyState

@export var chase_speed: float = 100.0

func physics_update(delta: float) -> void:
    var target = blackboard.target
    if target:
        var dir = sign(target.global_position.x - enemy.global_position.x)
        enemy.change_direction(dir)
        enemy.velocity.x = dir * chase_speed
```

### ES_Attack（攻击）

- 导出变量：
  - `attack_range: float = 100.0`
  - `move_speed: float = 0.0`
  - `cooldown: float = 3.0`
  - `attack_area: AttackArea`
- `can_attack()`：检查距离与冷却是否满足。
- `enter()`：
  - 播放攻击动画。
  - `duration = animation_player.current_animation_length`
  - 重置计时器。
  - 设置 `blackboard.can_decide = false`。
  - 启动冷却计时器（使用 `create_timer` + `await`）。
- `physics_update()`：
  - 计时器推进。
  - 计时结束退出状态。
- `exit()`：
  - `blackboard.can_decide = true`

### 攻击区域翻转

```gdscript
if attack_area:
    attack_area.flip(blackboard.dir)
```


## 5. 攻击区域（AttackArea）激活

- 给 `Living Armor` 添加 `AttackArea` 节点。
- 在攻击动画的关键帧调用 `AttackArea.activate(duration)`。
- 在 `AnimationPlayer` 中添加**方法调用轨道**：
  - 轨道目标：`AttackArea`
  - 方法：`activate`
  - 参数：`0.2`（秒）


## 6. 决策引擎 `DecisionEngine_BasicAttack`

```gdscript
func decide() -> EnemyState:
    if should_die(): return death_state
    if blackboard.damage_source: return stun_state
    if not blackboard.can_decide: return null
    if blackboard.target:
        if attack_state.can_attack():
            return attack_state
        return chase_state
    if blackboard.edge_detected:
        enemy.change_direction(-blackboard.dir)
    return walk_state
```

- 所有状态通过导出变量分配，便于混合搭配。


## 7. 调试与调参

- `EdgeSensor` 若因地势起伏误报，可：
  - 把射线起点抬高。
  - 开启 `collide_with_interior`。
- 攻击动画若卡死，通常是因为没有冷却，导致反复进入同一状态但动画未重置。
- 给 `ES_Attack` 加冷却可解决此问题。


## 8. 扩展方向

- 给攻击状态加 `move_speed_curve`，让敌人在攻击过程中动态前冲/后退。
- 为不同敌人配置不同的攻击范围、冷却、移动曲线。
