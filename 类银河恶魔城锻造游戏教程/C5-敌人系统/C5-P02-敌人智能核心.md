# C5-P02：敌人智能核心

> 本集搭建敌人系统的核心框架：通用敌人脚本、黑板（Blackboard）、决策引擎（Decision Engine）、敌人状态机（Enemy State Machine）和敌人状态（Enemy State）模板。


## 1. 新系统架构

```
Enemy (CharacterBody2D)
├── Sprite2D
├── AnimationPlayer
├── DamageArea
├── HazardArea
├── EdgeSensor
├── EnemyHitParticles
├── EnemyStateMachine
│   ├── ES_Walk
│   ├── ES_Stun
│   └── ES_Death
├── DecisionEngine
└── Blackboard (Resource，脚本内创建实例)
```


## 2. 通用敌人脚本 `Enemy`

### 信号

| 信号 | 参数 | 说明 |
|------|------|------|
| `direction_changed` | `new_direction: float` | 方向改变时发出 |
| `hit` | `attack_area: AttackArea` | 受到伤害时发出 |
| `killed` | — | 死亡时发出 |

### 导出变量

- `health: float`：最大生命值。
- `gravity_enabled: bool`：是否受重力影响。
- `face_left_at_start: bool`：初始面向左。
- `death_audio: AudioStream`：死亡音效。

### 关键机制

- 工具脚本（`@tool`），提供自定义配置警告。
- 通过 `get_children()` 自动收集根层级的 `AnimationPlayer`、`Sprite2D`、`DamageArea`、`HazardArea`、`EnemyStateMachine`、`DecisionEngine`。
- 运行时创建 `Blackboard` 资源实例，并同步生命值。
- 物理流程：
  1. 调用 `state_machine.change_state(decision_engine.decide())`。
  2. 调用 `state_machine.physics_update(delta)`。
  3. 若启用重力，施加 `get_gravity() * delta`。

### 配置警告

- 若缺少 `AnimationPlayer`、`Sprite2D`、`DamageArea`、`HazardArea`、`EnemyStateMachine`、`DecisionEngine`，在编辑器中显示警告。


## 3. 黑板 `Blackboard`

- 继承 `Resource`。
- 作用：集中存储敌人状态数据，供决策引擎、状态、传感器访问。

| 变量 | 类型 | 说明 |
|------|------|------|
| `health` | `float` | 当前生命值 |
| `target` | `Node2D` | 目标（通常是玩家） |
| `distance_to_target` | `float` | 与目标距离，无目标时为 `1` |
| `can_decide` | `bool` | 是否允许决策引擎决策 |
| `edge_detected` | `bool` | 是否检测到边缘 |
| `damage_source` | `AttackArea` | 最近一次伤害来源 |
| `dir` | `float` | 当前面向方向 |


## 4. 决策引擎 `DecisionEngine`

- 继承 `Node`。
- 职责：从黑板读取信息，返回下一个应进入的 `EnemyState`。
- 关键函数：
  - `decide() -> EnemyState`：默认返回 `null`。
  - `await super().ready`：等待父类就绪完成后再执行自定义逻辑。


## 5. 敌人状态 `EnemyState`

- 继承 `Node` 的状态模板。
- 每个状态有：
  - `animation_name: String`：可选动画名。
  - `enemy: Enemy`、 `state_machine: EnemyStateMachine`、 `blackboard: Blackboard`。
- 可重写函数：
  - `enter()`
  - `exit()`
  - `reenter()`
  - `physics_update(delta)`


## 6. 敌人状态机 `EnemyStateMachine`

- 继承 `Node`。
- 持有状态数组，记录当前状态与前一个状态。
- `setup(enemy, blackboard)`：初始化所有子状态的引用。
- `change_state(new_state)`：
  - 若 `new_state` 为空 → 返回。
  - 若与当前状态相同 → 调用 `reenter()`。
  - 否则退出当前状态，进入新状态。
  - 通知决策引擎当前状态已更新。
- `physics_update(delta)`：调用当前状态的 `physics_update`。


## 7. 代码模板（自定义脚本模板）

- 项目根目录下新建 `script_templates/` 文件夹。
- 分别创建 `DecisionEngine/default.gd` 与 `EnemyState/default.gd` 模板。
- 模板中可加入 `_meta_name_`、`_meta_description_`、`_meta_default_` 元数据。
