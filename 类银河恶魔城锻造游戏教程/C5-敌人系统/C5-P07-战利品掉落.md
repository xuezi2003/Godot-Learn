# C5-P07：战利品掉落

> 本集实现两个系统：
> 1. `HealthPickup`：可拾取的生命恢复道具。
2. `LootDropper` + `LootData`：敌人死亡时按概率掉落物品。


## 1. HealthPickup 生命恢复道具

### 节点结构

```
HealthPickup (CharacterBody2D)
├── Sprite2D
├── AnimationPlayer         # 心跳动画，自动播放、循环
├── CollisionShape2D        # 与地面/平台碰撞
└── Area2D                  # 检测玩家拾取
    └── CollisionShape2D
```

### 脚本要点

```gdscript
class_name HealthPickup
extends CharacterBody2D

@export var heal_amount: float = 10.0
@export var bounce_count: int = 8
@export var bounciness: float = 0.5

@onready var pickup_audio: AudioStream = preload("...")
```

### 物理弹跳

- 使用 `move_and_collide()` 代替 `move_and_slide()`。
- 碰撞时用 `velocity.bounce(collision.get_normal())` 反弹。
- 每次反弹减少 `bounce_count`，并乘以 `bounciness` 衰减。

```gdscript
func _physics_process(delta: float) -> void:
    if bounce_count > 0:
        velocity += get_gravity() * delta
        var collision = move_and_collide(velocity * delta)
        if collision:
            bounce_count -= 1
            velocity = velocity.bounce(collision.get_normal()) * bounciness
```

### 拾取处理

- `Area2D.body_entered` 触发。
- 确认是玩家后：`player.hp += heal_amount`
- 播放音效并 `queue_free()`。


## 2. 通用 Pickup 基类（直播/后续补充）

- 把 `HealthPickup` 与 `Coin` 的共同逻辑抽象为 `Pickup` 基类。
- 子类只需实现 `on_pickup(player)`。
- 基类处理：
  - 弹跳物理
  - 玩家检测
  - 音效播放
  - 防止重复拾取


## 3. LootData 资源

```gdscript
class_name LootData
extends Resource

@export_file("*.tscn") var item: String   # 掉落物场景路径
@export var min_amount: int = 1
@export var max_amount: int = 1
@export_range(0.0, 1.0) var drop_chance: float = 1.0
```


## 4. LootDropper 战利品掉落器

### 节点结构

```
LootDropper (Marker2D)
```

### 导出变量

```gdscript
@export var items: Array[LootData]
```

### 触发机制

- 所有者是 `Enemy` 时，连接 `owner.killed` → `_drop_loot()`。
- 所有者是可破坏物时，连接 `destroyed` 信号（需扩展）。

### 掉落逻辑

```gdscript
func _drop_loot() -> void:
    for loot in items:
        if randf() > loot.drop_chance:
            continue
        var count = randi_range(loot.min_amount, loot.max_amount)
        var scene = load(loot.item)
        for i in count:
            var drop = scene.instantiate()
            owner.add_sibling.call_deferred(drop)
            drop.global_position = global_position
            if drop is CharacterBody2D:
                drop.velocity = Vector2(
                    randf_range(-50, 50),
                    randf_range(-400, -200)
                )
```


## 5. 掉落概率说明

- `drop_chance = 1.0`：必掉。
- `drop_chance = 0.25`：约 25% 概率掉落。
- 每个 `LootData` 独立判定。


## 6. 配置示例

| 敌人类型 | 掉落物 | 最小 | 最大 | 概率 |
|----------|--------|------|------|------|
| 史莱姆 | HealthPickup | 1 | 2 | 0.25 |
| 箱子 | HealthPickup | 1 | 1 | 1.0 |
| 骷髅 | Coin | 5 | 10 | 1.0 |


## 7. 注意事项

- 实例化节点需在 `call_deferred` 中进行，因为此时正在响应信号。
- 掉落物位置建议在敌人头部上方（`LootDropper` 的位置）。
- 可进一步扩展为基于资源的库存系统。
