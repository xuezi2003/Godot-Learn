# C5-P05-2：让敌人能听到玩家的声音

> 本集在 `PlayerSensor` 中加入听觉探测功能：玩家发出声音时，附近敌人会根据距离和音量判断是否“听到”玩家，并短暂将其设为目标。


## 1. 设计思路

- 不分析真实音频波形，而是“假装”有声波传播：
  - 每个玩家音效带一个“音量”值。
  - 听觉探测器比较音量、距离和敌人的听觉敏感度。
- 与视觉探测器共用同一个节点，简化管理。


## 2. 扩展 `PlayerSensor`

### 新增导出变量

```gdscript
@export var use_audio_sensor: bool = true
@export var audio_detection_distance: float = 512.0
@export var min_audio_perception: float = 0.25   # 听觉阈值
```

### 新增信号

```gdscript
signal player_sound_heard(position: Vector2, volume: float)
```


## 3. 全局音频系统改造

- 在 `AudioManager.play_sound_at_position` 中新增可选参数：
  ```gdscript
  func play_sound_at_position(audio: AudioStream, position: Vector2,
                              ignore_pool: bool = false,
                              from_player: bool = false,
                              volume: float = 0.5) -> void
  ```
- 若 `from_player == true`，广播信号：
  ```gdscript
  MessageBus.player_sound_made.emit(position, volume)
  ```

### 玩家状态中的音效调用示例

| 动作 | 音量参考 |
|------|----------|
| 跳跃 | 0.25 |
| 落地 | 0.5 |
| 挥剑 | 0.5 |
| 地面猛击 | 0.75 |
| 冲刺 | 0.75 |


## 4. 听觉判断逻辑

```gdscript
func _on_player_sound_heard(sound_position: Vector2, volume: float) -> void:
    if not use_audio_sensor:
        return

    var sound_distance = global_position.distance_to(sound_position)
    if sound_distance > audio_detection_distance:
        return

    var distance_ratio = 1.0 - (sound_distance / audio_detection_distance)
    var perceived_volume = volume * distance_ratio * 2.0

    if perceived_volume >= min_audio_perception:
        enemy.blackboard.target = get_tree().get_first_node_in_group("player")
        search_timer = search_duration
```


## 5. 调试与调参

- 打印 `distance`、`distance_ratio`、`perceived_volume` 帮助调参。
- 听觉敏感度越高（`min_audio_perception` 越低），敌人越容易被惊动。
- 不同敌人可配置不同听觉参数：
  - 史莱姆：迟钝（`0.5`）
  - 狼/盗贼：敏锐（`0.1–0.2`）


## 6. 注意事项

- 听觉与视觉共享 `blackboard.target`，可能互相覆盖；目前方案够用。
- 更复杂的潜行游戏可引入单独的“警觉等级”或“搜索状态”。
- 这是一个“伪物理”系统，适合平台动作游戏，不必追求真实声学模拟。
