# C5-P03：更优秀的敌人

> 本集把上集搭建的核心框架跑通，用新的状态机 + 决策层系统重建史莱姆，并实现行走、眩晕、死亡三个基础状态，以及一个会随敌人方向自动翻转的 `EdgeSensor`。


## 1. 创建脚本模板

- 在 `script_templates/` 下创建：
  - `DecisionEngine/default.gd`
  - `EnemyState/default.gd`
- 模板好处：新建脚本时自动继承正确类，避免命名冲突与重复代码。


## 2. 新的史莱姆场景 `Slime2`

```
Slime2 (Enemy)
├── Sprite2D
├── AnimationPlayer
├── CollisionShape2D
├── DamageArea
├── HazardArea
├── EdgeSensor
├── EnemyStateMachine
│   ├── ES_Walk        # 默认状态
│   ├── ES_Stun
│   └── ES_Death
├── BasicDecisionEngine
└── EnemyHitParticles（可选）
```


## 3. 决策引擎 `BasicDecisionEngine`

- 继承 `DecisionEngine`。
- 使用 `@onready var` 缓存行走、眩晕、死亡状态引用。
- 决策顺序：
  1. 死亡 → 返回死亡状态。
  2. 无法决策（`can_decide == false`）→ 返回 `null`。
  3. 有伤害来源 → 返回眩晕状态。
  4. 检测到边缘 → 转身。
  5. 默认 → 返回行走状态。


## 4. 状态实现

### ES_Walk

- `enter()`：播放行走动画（动画名可选）。
- `physics_update(delta)`：
  - `velocity.x = walk_speed * blackboard.dir`
  - 撞墙时调用 `enemy.change_direction(-blackboard.dir)`。

### ES_Stun

- 变量：
  - `knockback_speed: float`（导出）
  - `timer: float`、`duration: float`、`velocity_x: float`
- `enter()` / `reenter()` → `start()`：
  - 播放眩晕动画。
  - 若已在播放则 `seek(0)` 重新开始。
  - `duration = animation_player.current_animation_length`
  - 重置 `timer = 0`
  - 计算击退方向与速度
  - 清除 `blackboard.damage_source`
  - `blackboard.can_decide = false`
- `physics_update(delta)`：
  - `timer += delta`
  - 击退速度按 `(1 - timer / duration)` 衰减
  - 计时结束后 `blackboard.can_decide = true`
- `exit()`：确保 `can_decide = true`

### ES_Death

- 类似眩晕，但无需 `reenter`。
- `enter()`：
  - 播放死亡动画与死亡音效。
  - 清除伤害来源，禁用决策。
  - `await animation_finished` 后 `queue_free()`。
- > 死亡音效原本以导出变量形式放在通用 `Enemy` 脚本上，本集为了简化敌人脚本、让死亡逻辑自包含，改为由 `ES_Death` 状态持有并播放。


## 5. 传感器 `EdgeSensor`（升级版）

- 继承 `RayCast2D`。
- 自动检测所有者是 `Enemy`。
- 连接 `enemy.direction_changed` 信号，随方向翻转位置。
- 不在地面上时不工作。
- 碰撞状态变化时更新 `blackboard.edge_detected`。
- 无需再手动把信号连到史莱姆脚本，实现完全解耦。


## 6. 关键细节

- 默认状态是状态机子节点列表中的**第一个**。
- 攻击来源在 `Enemy._on_damage_taken(attack_area)` 中写入黑板：
  ```gdscript
  blackboard.damage_source = attack_area
  ```
- 方向与生命值全部迁移到黑板，敌人脚本本身不再直接保存。

## 7. 修复 Player 脚本警告

- 问题：Player 的 `_on_damage_taken(attack_area: AttackArea)` 参数名与类成员变量 `attack_area` 同名，产生**遮蔽（shadowing）**警告。
- 修复：将参数重命名为 `a` 或其他不冲突的名字，函数体内同步替换。
- 意义：避免敌人攻击玩家时，错误地引用玩家自身攻击区域而非敌人的攻击区域。
