# C5-P05-1：敌人玩家探测器

> 本集实现一个可复用的 `PlayerSensor`（玩家探测器/视野锥），让敌人能检测玩家进入/离开视野，并计算与目标的距离，为后续的追逐、攻击状态提供数据。


## 1. 设计目标

- 检测玩家何时进入/离开视野范围。
- 计算并更新到玩家的距离。
- 提供“搜索计时器”：玩家离开后不会立刻忘记，而是继续追踪一段时间。
- 完全模块化，插入任意敌人即可工作。


## 2. 场景结构

```
PlayerSensor (Area2D)
└── CollisionPolygon2D    # 三角形视野锥
```


## 3. 脚本 `PlayerSensor`

### 继承与变量

```gdscript
class_name PlayerSensor
extends Area2D

signal player_entered
signal player_exited
signal search_started

@export var search_duration: float = 2.0

var enemy: Enemy
var search_timer: float = 0.0
```

### 视野锥配置

- 在 `_ready()` 中清空所有 Layer/Mask。
- 若所有者是 `Enemy`：
  - `enemy = owner`
  - 设置 Mask 为玩家层（第 5 层）。
  - 连接 `body_entered`、`body_exited`。
  - 连接 `enemy.direction_changed`，随方向翻转 `scale.x`。

### 多边形顶点示例

```gdscript
# 局部坐标
Vector2(0, 0)
Vector2(256, -128)
Vector2(256, 128)
```


## 4. 关键逻辑

### 玩家进入

```gdscript
func _on_body_entered(body: Node2D) -> void:
    player_entered.emit()
    enemy.blackboard.target = body
```

### 玩家离开

- 不立即清除目标。
- 启动 `search_timer = search_duration`。
- 在 `search_duration` 内若玩家重新进入，重置计时器为 0。
- 计时器归零后才清除 `blackboard.target`。

### 修复的 Bug

- 问题：玩家离开视野后立刻重新进入，计时器仍在倒计时，最终仍会清除目标。
- 修复：在 `_on_body_entered` 中把 `search_timer = 0`。


## 5. 距离计算

- 在 `Enemy._physics_process` 中调用：
  ```gdscript
  blackboard.update_distance_to_target(global_position)
  ```
- `Blackboard` 中的实现：
  - 有目标：`distance_to_target = global_position.distance_to(target.global_position)`
  - 无目标：`distance_to_target = 1`


## 6. 调试技巧

- 可在场景中临时添加 `Label` 显示 `target` 名称与 `distance_to_target`。
- 打开 `Visible Collision Shapes` 查看视野锥范围。


## 7. 后续扩展方向

- 在黑板中增加 `searching` 布尔值，实现“搜索状态”。
- 可创建不同类型的探测器（听觉、嗅觉等），通过黑板统一交互。
