# C2-P5：关卡场景切换系统-1

> 创建可复用的关卡过渡门场景，配置 Area2D 检测、编辑器内预览与导出变量，为全局 SceneManager 做准备。

## 1. 前置准备

- 至少准备 2~3 个关卡场景，并确保它们在过渡处大致对齐。
- 示例连接：
  - 第一关右侧 → 第二关左侧（水平过渡）
  - 第二关底部 → 第三关顶部（垂直过渡）
  - 第三关顶部 → 第二关底部（返回）

## 2. 玩家碰撞层

- 将玩家分配到碰撞层 **5**，并命名为“玩家”层。
- 其他对象不占用该层，过渡门只检测此层。

## 3. 关卡过渡门场景结构

```
LevelTransition (Node2D)
└── Area2D
    └── CollisionShape2D (RectangleShape2D)
```

## 4. Area2D 碰撞设置

| 属性 | 值 | 说明 |
|------|-----|------|
| Collision Layer | 全部关闭 | 不处于任何层 |
| Collision Mask | 仅第 5 层 | 只检测玩家 |
| Monitoring | 开启 | 检测进入的物体 |
| Monitorable | 关闭 | 不被其他区域检测 |

## 5. 碰撞形状定位

- 形状大小设为 **32×32**。
- 偏移设置为 `(16, -16)`，使方框的左下角与节点原点重合。
- 这样便于后续根据过渡方向正确翻转/缩放。

## 6. 脚本导出变量

```gdscript
@tool
@icon("res://general/icons/level_transition_icon.svg")
class_name LevelTransition
extends Node2D

enum Side { LEFT, RIGHT, TOP, BOTTOM }

@export_range(2, 12, 1, "or_greater") var size: int = 2:
    set = _set_size
@export var location: Side = Side.LEFT:
    set = _set_location
@export_file("*.tscn") var target_level: String = ""
@export var target_area_name: String = "LevelTransition"
```

- `size`：过渡门高度/宽度（以瓦片网格数为单位）。
- `location`：过渡门位于关卡的哪一侧（左/右/上/下）。
- `target_level`：目标关卡场景文件路径。
- `target_area_name`：目标关卡中对应的过渡门名称，用于生成玩家。

## 7. 编辑器内预览

- 通过 `apply_area_settings()` 根据 `size` 与 `location` 缩放/翻转 `Area2D`：
  - 左侧/右侧：按 `size` 缩放 Y，X 缩放 `1`（左）或 `-1`（右）。
  - 顶部/底部：按 `size` 缩放 X，Y 缩放 `1`（顶）或 `-1`（底）。
- 为 `Area2D` 设置调试颜色，便于在编辑器中查看过渡范围。

## 8. 运行时行为

- 在游戏运行时连接 `Area2D.body_entered` 信号。
- 当玩家进入时，调用全局 `SceneManager` 处理切换，而不是在过渡门内直接调用 `change_scene_to_file`。
- 直接切换会销毁玩家并触发物理回调错误；由 `SceneManager` 统一加载/卸载更安全。

## 9. 为什么不直接切换场景

- 直接调用 `get_tree().change_scene_to_file(target_level)` 会：
  - 销毁当前玩家实例。
  - 新关卡若没有玩家节点则玩家消失。
  - 在物理回调中释放场景会报“non-physics removal”错误。
- 因此需要一个全局管理器保存玩家、控制加载与转场动画。

## 10. 课后任务

- 为每个关卡添加过渡门，设置目标关卡与目标区域名。
- 尝试调整 `size` 与 `location`，观察编辑器内预览框的变化。
- 准备多个关卡的对齐关系，为下一集 SceneManager 实现打好基础。
