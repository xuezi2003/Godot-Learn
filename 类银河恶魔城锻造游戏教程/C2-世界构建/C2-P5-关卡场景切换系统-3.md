# C2-P5：关卡场景切换系统-3

> 创建全局 SceneManager，实现带方向性擦除淡入淡出动画的关卡切换，并在切换期间暂停游戏以保持玩家位置正确。

## 1. SceneManager 的定位

- 将 `SceneManager` 设为 **AutoLoad**（全局单例）。
- 其 `process_mode` 必须设为 **Always**，因为切换时会暂停游戏树，而它仍需执行补间动画与加载逻辑。

## 2. 转场 UI 结构

- 使用一个占满屏幕的 `Control` 节点（如 `Fade`）。
- 内部放置 `TextureRect`，使用 `GradientTexture2D`：
  - 中心纯黑，向四周渐变为透明。
  - 矩形尺寸设为视口的 3 倍，保证向任意方向擦除时都能覆盖屏幕。
  - 渐变黑色停止点设在 0.333 左右，使黑场、过渡、透明三部分各占一个屏幕。
  - > 作者最初尝试将黑色停止点设为 0.5，但后来为了配合 3 倍视口尺寸、让三个区域等大，调整为 0.333。

## 3. 方向性擦除位置

```gdscript
func get_fade_position(dir: String) -> Vector2:
    var offset := Vector2(480 * 2, 270 * 2)
    match dir:
        "left":   return offset * Vector2(-1, 0)
        "right":  return offset * Vector2(1, 0)
        "top":    return offset * Vector2(0, -1)
        "bottom": return offset * Vector2(0, 1)
    return Vector2.ZERO
```

- `dir` 来自 `LevelTransition` 的 `location` 枚举（小写）。
- 偏移量为两倍视口尺寸，确保起始位置在屏幕之外。

## 4. 补间动画

```gdscript
func fade_screen(from: Vector2, to: Vector2) -> Signal:
    fade.visible = true
    fade.position = from
    var tween := create_tween()
    tween.tween_property(fade, "position", to, 0.2)
    return tween.finished
```

## 5. 切换流程

1. 接收目标关卡、进入方向、目标区域名。
2. `get_tree().paused = true`
3. `await fade_screen(fade_pos, Vector2.ZERO)` —— 屏幕擦黑。
4. 加载新场景（`change_scene_to_file` 或 `change_scene_to_packed`）。
5. 等待新场景准备就绪的信号。
6. 将玩家移动到新关卡中对应名称的 `LevelTransition` 位置。
7. `await fade_screen(Vector2.ZERO, -fade_pos)` —— 向相反方向擦除打开。
8. `fade.visible = false`，`get_tree().paused = false`。

## 6. 暂停的重要性

- 在 0.2 秒的淡入 + 0.2 秒淡出期间，若不暂停游戏，玩家会继续移动，导致切换后位置偏移或卡进地形。
- 将 `SceneManager` 的 `process_mode` 设为 `ALWAYS`，可确保暂停期间补间动画与加载逻辑仍然运行。

## 7. 细节优化

- 若垂直向上过渡时玩家跳不上去，可在进入过渡门时给玩家一个向上的速度加成。
- 转场层默认隐藏，仅在切换时显示。
- 加载耗时较长时，画面会保持黑屏，直到新场景准备就绪再打开，避免中间暴露半成品画面。

## 8. 与 LevelTransition 的协作

- `LevelTransition` 负责：检测玩家、提供方向/目标关卡/目标区域名。
- `SceneManager` 负责：暂停、淡入淡出、加载新场景、定位玩家、恢复游戏。
- 两者解耦，便于后续扩展存档/读档、返回主菜单、Boss 战转场等功能。
