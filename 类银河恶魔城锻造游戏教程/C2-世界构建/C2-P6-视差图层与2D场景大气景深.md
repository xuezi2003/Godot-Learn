# C2-P6：视差图层与 2D 场景大气景深

> 使用 Parallax2D 与多个 TileMapLayer 构建多层背景/前景，通过滚动缩放、色彩调制与雾气渐变营造景深与氛围。

## 1. Parallax2D 基础

- Godot 4 中新增 **Parallax2D** 节点，用于替代旧版 ParallaxBackground。
- 关键属性：**Scroll Scale**（滚动比例）。
  - `1.0`：与玩家同速移动。
  - `0.0`：完全固定，适合天空或远山。
  - `0.5`：以一半速度移动，营造中景深度。
  - `>1.0`：比玩家移动更快，适合前景元素。
- 可分别设置 X 与 Y 轴的滚动比例；对横版卷轴通常 Y 保持 `1.0`。

## 2. 多层景深规划

| 层级 | 滚动比例 | 色彩/处理 | 作用 |
|------|----------|-----------|------|
| 天空 | 0.0 | 跟随摄像机，通常用纯色或渐变 | 最远背景 |
| 远景树/山 | 0.2 ~ 0.4 | 低对比度、偏蓝/紫、偏亮 | 距离感 |
| 中景树林 | 0.5 ~ 0.7 | 中等明度、偏青 | 过渡深度 |
| 近景装饰 | 0.8 ~ 0.9 | 稍深、细节更多 | 贴近玩家 |
| 主地形 | 1.0 | 正常绘制 | 可玩区域 |
| 前景剪影 | 1.3 ~ 1.6 | 深色，Z Index 高于玩家 | 框架与层次 |

## 3. 组织场景树

- 将背景视差层、关卡瓦片、道具、前景视差层分别用空 `Node2D` 节点分组。
- 示例结构：

```
Level
├── BackgroundParallax
│   ├── Sky
│   ├── FarTrees
│   ├── MidTrees
│   └── Mist
├── LevelTilemaps
│   ├── MainGround
│   └── OneWayPlatforms
├── Props
│   ├── Plants
│   └── Torches
├── Player
└── ForegroundParallax
    ├── ForegroundTrees
    └── Vignette
```

## 4. 背景层实现要点

- 每个 Parallax2D 子节点可放置 TileMapLayer、Sprite2D 等任意视觉节点。
- 背景 TileMapLayer 应**关闭碰撞检测**，避免玩家撞到背景物体。
- 使用 **Modulate** 为瓦片统一着色，使其融入远景。
- 远处物体应保持低对比度，减少高光与深黑阴影。

## 5. 色彩与大气透视

- 现实世界中，远处物体会沾染天空色调（蓝/紫）。
- 在像素游戏中可用纯色调制模拟：越远越偏蓝、偏亮、越接近天空色。
- 在中间层加入半透雾气渐变精灵，可进一步压平远景对比度。

## 6. 前景层实现要点

- 前景层滚动比例设为 `1.4` 或更高，并设置较高的 **Z Index**（如 10），确保覆盖玩家。
- 使用深色剪影或深绿色，避免遮挡玩法区域。
- 可用前景物体遮挡瓦片底部边缘，避免“漂浮”感。

## 7. 常用操作

- 复制 Parallax2D + TileMapLayer 组合，然后调整滚动比例、位置与颜色，可快速生成多层。
- 对“重复”纹理的视差层，可设置 **Repeat Size** 与 **Repeat Count** 实现无限平铺。
- 用 `z_index` 控制玩家、背景、前景的绘制顺序。

## 8. 课后任务

- 为第一关添加至少 4 层视差背景 + 1 层前景。
- 为森林、城堡、地牢分别设计不同的远景配色。
- 保持远处图层简单，避免过度细节分散玩家注意力。
