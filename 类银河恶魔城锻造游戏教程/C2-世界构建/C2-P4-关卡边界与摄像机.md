# C2-P4：关卡边界与摄像机

> 为玩家添加 Camera2D，并创建可复用的 LevelBounds 节点，在编辑器中可视化边界框并限制相机活动范围。

## 1. 为玩家添加摄像机

1. 打开玩家场景。
2. 添加 **Camera2D** 节点作为玩家子节点。
3. 将摄像机稍微向上移动，使画面中心聚焦在角色头部附近。
4. 摄像机作为子节点会自动跟随玩家，无需额外脚本。

## 2. 关卡边界节点 LevelBounds

### 文件组织

```
res://general/
├── icons/
│   └── level_bounds_icon.svg
└── scripts/
    └── level_bounds.gd
```

### 脚本要点

```gdscript
@tool
@icon("res://general/icons/level_bounds_icon.svg")
class_name LevelBounds
extends Node2D

@export_range(480, 2048, 32, "suffix:px") var width: int = 480:
    set = _on_width_changed
@export_range(270, 2048, 32, "suffix:px") var height: int = 270:
    set = _on_height_changed

func _ready():
    z_index = 256
    if Engine.is_editor_hint():
        return
    _setup_camera_limits()

func _draw():
    if not Engine.is_editor_hint():
        return
    var r := Rect2(Vector2.ZERO, Vector2(width, height))
    draw_rect(r, Color(0.0, 0.45, 1.0, 0.6), false, 3.0)
    draw_rect(r, Color(0.0, 0.75, 1.0, 1.0), false, 1.0)

func _on_width_changed(new_width: int) -> void:
    width = new_width
    queue_redraw()

func _on_height_changed(new_height: int) -> void:
    height = new_height
    queue_redraw()

func _setup_camera_limits() -> void:
    var camera: Camera2D
    while camera == null:
        await get_tree().process_frame
        camera = get_viewport().get_camera_2d()
    camera.limit_left = int(global_position.x)
    camera.limit_top = int(global_position.y)
    camera.limit_right = int(global_position.x) + width
    camera.limit_bottom = int(global_position.y) + height
```

### 关键说明

- `@tool` 让脚本在编辑器中运行，可实时绘制预览框。
- `@export_range` 限制宽高最小值为视口分辨率，步长 32 与瓦片网格对齐。
- `z_index = 256` 保证预览框始终在最上层显示。
- `_draw()` 仅在编辑器中执行，游戏运行时不绘制。
- 通过 `get_viewport().get_camera_2d()` 获取当前活动相机；由于相机可能在关卡加载后稍晚就绪，使用 `while` + `await` 等待。

## 3. 使用方式

1. 在关卡场景中添加 `LevelBounds` 节点。
2. 将其拖动到关卡左上角（可开启网格吸附）。
3. 调整 `width` / `height` 使其覆盖整个关卡。
4. 运行游戏，摄像机会被限制在蓝色框范围内。

## 4. 优势

- 边界与瓦片地图解耦，可灵活控制可见区域。
- 不需要为了 camera limit 而在边缘多铺墙或平台。
- 方便上下左右任意方向留出过渡入口。
