# 06 数组、循环与字典

## 数组

当一个变量需要存储**一整列元素**时，用**数组**。

### 定义与类型

```gdscript
var items = []                                  # 空数组
var items = ["potion", "feather", "stolen harp"] # 带元素
```

GDScript 允许在数组中**混合数据类型**。若希望限制为特定类型，可静态定义：

```gdscript
var items: Array[String] = ["potion", "feather", "stolen harp"]
```

此时放入非字符串（如数字 `3`）会报错。

### 用索引访问与修改

元素按加入顺序自动分配索引，**从 0 开始**：`potion` → 0，`feather` → 1，`stolen harp` → 2。

```gdscript
print(items[0])            # potion
items[1] = "smelly sock"   # 修改第二个元素
```

### 常用函数

```gdscript
items.remove_at(1)                 # 删除指定索引处的元素
items.append("overpowered sword")  # 在末尾追加新元素
```

## 循环

循环让我们把代码重复执行多次，非常适合遍历数组。

### for 遍历数组

```gdscript
for item in items:
    print(item)

for item in items:
    if item.length() > 6:   # item 的长度（字符数）大于 6 才打印
        print(item)
```

### for 运行指定次数

```gdscript
for n in 8:                 # n 从 0 到 7，共运行 8 次
    print(n)
```

`n` 是循环的当前迭代次数。

### while 循环

只要条件满足就一直重复：

```gdscript
var glass = 0.0
while glass < 0.5:
    glass += randf_range(0.01, 0.2)
    print(glass)
print("杯子现在已经半满了！")
```

> ⚠️ 使用 `while` 时要非常小心**无限循环**：如果条件永远无法变为假（比如忘了在循环内改变判断变量），程序会卡死并产生溢出错误。

### break 与 continue

- `break`：立即**跳出**循环，继续执行循环之后的代码。
- `continue`：立即跳到循环的**下一次迭代**。

```gdscript
while glass < 0.5:
    glass += randf_range(0.01, 0.2)
    if glass > 0.2:
        break               # 提前结束循环
    print(glass)
```

## 字典

数组适合存有序列表，但用索引访问有时不够直观；这时**字典**更合适。字典保存**键值对（key-value）**：就像现实中的字典，「键」是要查的单词，「值」是它的定义。

### 定义与访问

```gdscript
var players = {
    "Crook": 1,
    "Villain": 35,
    "Boss": 100,
}
print(players["Villain"])      # 35
```

> 在最后一个条目后加逗号是个好习惯，方便之后添加新条目。

### 修改与新增

```gdscript
players["Villain"] = 50        # 重新赋值
players["Dwayne"] = 999        # 添加新条目
```

### 用 for 遍历字典

遍历字典会得到所有的**键**：

```gdscript
for username in players:
    print(username + ": " + str(players[username]))
```

### 嵌套字典

字典的键和值都可以是多种类型，还能**嵌套**数组或字典。例如同时存储每个玩家的等级和生命值：

```gdscript
var players = {
    "Crook": {"level": 1, "health": 80},
    "Boss": {"level": 100, "health": 500},
}
print(players["Boss"]["health"])   # 用两个键访问 → 500
```

这样就能开始为游戏数据建模：玩家状态、道具、增益效果等。
