# 10 继承与组合

## 继承

**继承**指从一个类**派生**出另一个类的能力。其实我们一直在用它——每个脚本顶部都有 `extends`：

```gdscript
extends Node
```

这表示脚本从 `Node` 类派生，于是 `Node` 类的**所有函数和变量在脚本中都可用**。

### Godot 节点层级即继承

添加新节点时，可以看到一些节点按层级组织在另一些节点之下，这正是因为它们**继承**自上层节点。例如 `AnimatedSprite2D` 和 `Camera2D` 都继承自 `Node2D`——因为 `Node2D` 是 2D 空间中一切存在物的基础类，相机和精灵都需要一个位置。

用 `class_name` 定义的 `Character` 类也会出现在节点列表中，因为定义一个类本质上就是在**定义一种新的节点类型**，它继承自 `Node`。

### 确保扩展正确的类

大多数情况下，继承对我们的意义就是：**确保正在编写的脚本继承（扩展）了正确的类**。例如要控制玩家移动，就创建一个 `CharacterBody2D` 节点并添加脚本：

```gdscript
extends CharacterBody2D
# 自动扩展 CharacterBody2D，因而可以访问 velocity、move_and_slide() 等功能
```

## 组合

尽管 Godot 在节点上使用了继承，但通常还有**更好**的方式来组织代码结构——Godot 其实非常适合另一种称为**组合（composition）** 的方式。
