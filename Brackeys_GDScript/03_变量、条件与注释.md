# 03 变量、条件与注释

## 变量 1.0

**变量**是存储信息的容器。以游戏角色为例，可以用变量存储姓名、生命值、伤害，以及是否存活等信息。

### 声明与赋值

在脚本顶部用 `var` 声明变量，接着写名称，再赋一个默认值：

```gdscript
var health = 100

func _ready():
    print(health)      # 100
```

### 修改变量

```gdscript
health = 40            # 直接赋新值
health = 20 + 30       # 也可以赋一个计算结果 → 50
health += 20           # 加等于
health -= 10           # 减等于
health *= 2            # 乘等于
health /= 4            # 除等于
```

代码从上到下依次执行，最后打印结果。

### 结合输入修改变量

例如玩家受伤时扣血：

```gdscript
func _input(event):
    if event.is_action_pressed("my_action"):
        health -= 10
        print(health)
```

此时生命值可以一直降到 0 以下且没有任何后果——这就需要用 `if` 语句让游戏对变量做出反应。

## if 条件语句

`if` 语句是所有逻辑的「粘合剂」：只有条件满足，内部代码才会执行。

```gdscript
if health <= 0:
    health = 0
    print("YOU DIED!")
```

### 比较运算符

| 运算符 | 含义 |
|--------|------|
| `==` | 等于 |
| `!=` | 不等于 |
| `<` | 小于 |
| `>` | 大于 |
| `<=` | 小于等于 |
| `>=` | 大于等于 |

### 组合多个条件

- `and`：两个条件都满足才为真。
- `or`：两个条件满足其一即为真。

### else 与 elif

- `else`：条件为假时执行。
- `elif`（else if 的简写）：串联多个条件。

```gdscript
if health <= 0:
    health = 0
    print("YOU DIED!")
elif health < 50:
    print("你受伤了")
else:
    print("你很健康")
```

`if` 语句可以嵌套（分层）以获得越来越具体的功能，但**不要过度使用**，否则很容易在层层 `if` 中迷失。

## 注释

注释用来解释和记住代码「是什么」和「为什么」。用井号 `#` 添加，可以放在一行之上，也可以放在代码之后：

```gdscript
# 这是对下一行代码的说明
health = 100

health = 100  # 这是仅对该行的注释
```

### 用注释临时禁用代码

在代码前加 `#` 即可让它不被执行。**惯例**：注释掉代码时通常紧跟 `#` 不加空格，以表明其后跟随的是代码。

还可以选中多行，右击选择**切换注释（Toggle Comment）**批量处理。

> 注意：函数体不能完全空白，否则会报错，需要用 `pass` 关键字占位。
