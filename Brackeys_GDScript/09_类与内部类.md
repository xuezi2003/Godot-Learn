# 09 类与内部类

## 类

GDScript 是**面向对象**的语言：我们通常把代码组织成一个个相互交互的封闭对象，主要通过**类**来实现。

可以把类想象成一张**蓝图**。比如制作 RPG 需要许多角色，就创建一个「角色」类，定义所有角色共有的变量和逻辑：

- 变量：名字、生命值、对话等。
- 逻辑：交谈、死亡等函数。

然后用这个类创建**实例（instance）**——实例是类的具体版本。例如「药剂商人」实例有 50 点生命值和一句特定台词；其他实例有相同的变量但取值不同，且都能交谈、死亡。

> 其实你在 Godot 里已经接触过很多类了：**所有内置节点都是类**。节点是包含变量与逻辑的自包含对象，添加一个精灵节点就是在实例化精灵类。因此，创建一个脚本实际上也相当于在创建一个类。

### 用 class_name 定义类

新建一个节点（基础 `Node`）命名为 `character`，添加脚本。在顶部用 `class_name` 把脚本明确定义为一个类，并用 `@export` 让每个实例都能在检查器中设置属性：

```gdscript
# character.gd
class_name Character
extends Node

@export var profession: String
@export var health: int

func die():
    health = 0
    print(profession + " died")
```

保存后，在检查器中即可看到 `profession` 和 `health` 两个属性。复制该节点可创建多个实例（如「药剂商人」40 血、「前冒险家」30 血、「骑士」150 血），各自填入不同的值。

### 在别处调用实例

因为我们已经用 `class_name` 命名了类，可以直接用它作为导出变量的类型，在编辑器里拖入或分配实例，而不必担心路径变化：

```gdscript
# main.gd
@export var character_to_kill: Character

func _ready():
    character_to_kill.die()
```

把某个角色拖入 `main` 节点的槽位后运行，就会在该角色上调用 `die()`，例如打印 `knight died`。

## 内部类

**内部类**是定义在另一个类**内部**的类，主要用于捆绑一些变量（可能再加一两个函数）。它可以作为字典的良好替代，有时**更安全**。

```gdscript
class Equipment:
    var armor = 10
    var weight = 5

var chest = Equipment.new()    # new() 创建 Equipment 的实例
var legs = Equipment.new()

func _ready():
    chest.armor = 20
    print(chest.armor)         # 20
    print(legs.weight)         # 5
```

之所以说比字典安全，是因为 GDScript 知道 `Equipment` 类里有哪些变量：如果访问不存在的成员，会在**运行前**就报错，而不是等代码执行到那一行才出问题——这就是**类型安全**。
